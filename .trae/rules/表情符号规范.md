# 表情符号使用规范

## 使用原则

- 避免连续超过3个 emoji（视觉干扰）
- 技术文档中慎用夸张表情（如😂）
- 确保跨平台显示一致性（使用基础符号）
- 文字结尾与 emoji 之间的空格可加可不加，建议增加以提高阅读体验
- 保持礼貌，避免使用有攻击性或侮辱性的 emoji

## 场景推荐

| 场景 | 推荐 emoji |
|------|-----------|
| 章节标题 | 📚 📖 📝 📊 |
| 提示/警告 | 💡 🔔 ⚠️ ⛔ |
| 正确/成功 | ✅ ⭕ ✔️ |
| 错误/失败 | ❌ ❎ |
| 问题/疑问 | ❓ ❔ |
| 禁止 | ⛔ 🚫 |

## 文档位置

- **每个章节标题必须包含表情符号**（如"## 1. 概述 📚"）
- **每个二级和三级标题必须包含表情符号**
- **每个列表项可以添加表情符号**（非强制，避免过多）
- **每个表格标题可以添加表情符号**（视情况）
- **关键概念和重要提示必须包含表情符号**
- **段落中的重要句子可以添加表情符号**
- **代码示例的说明文字可以添加表情符号**
- **适度使用，避免过度**
