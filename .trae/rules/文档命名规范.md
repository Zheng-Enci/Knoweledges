# 文档命名规范 📝

## 基本原则

**文档主标题必须与文件名完全一致，且包含编号。**

文档名是文档的"身份证"，应该准确反映文档内容的边界和范围。命名过于宽泛会导致内容与名称不匹配，让读者产生错误预期。

## 具体规则

### 规则一：名称范围要与内容范围精确匹配

| 错误示例 | 问题 | 正确示例 |
|---------|------|---------|
| `03-深度学习.md` | 范围太广，一篇文档不可能讲完深度学习 | `03-梯度下降优化器.md` |
| `04-Transformer.md` | Transformer包含太多组件 | `04-残差连接与层归一化.md` |
| `05-注意力机制.md` | 可能包含多头、自注意力、交叉注意力等 | `05-缩放点积注意力代码实现.md` |

### 规则二：名称要足够具体，能一眼看出边界

好的文档名应该让读者一看就知道：
- 这是什么（不是泛泛的"机制"、"模型"）
- 边界在哪里（不是包含N种东西的"综述"）

| 场景 | 好的命名 | 说明 |
|------|---------|------|
| 只讲缩放点积，不讲多头 | `05-缩放点积注意力代码实现.md` | 明确限定在"缩放点积" |
| 只讲位置编码的原始方案 | `06-正弦位置编码原理.md` | 限定在"正弦"这一种 |
| 讲Transformer整体结构 | `07-编码器解码器架构.md` | 边界清晰 |

### 规则三：主标题与文件名必须完全一致

文件名：`04-缩放点积注意力代码实现.md`

正确主标题：
```markdown
# 04-缩放点积注意力代码实现 📚
```

错误主标题：
```markdown
# 缩放点积注意力实现 📚      <!-- 缺少编号 -->
# 04-缩放点积注意力 📚         <!-- 缺少"代码实现"，与文件名不一致 -->
```

### 规则四：编号规则

- 编号从01开始递增
- 同一系列文档使用连续编号
- 编号必须出现在文件名和主标题的最前面
- 编号格式：`01-`、`02-`、...、`10-`

## 判断标准

**提问检验法**：当你看到一个文档名时，能否准确说出：
1. 这篇文档**肯定包含**什么内容？
2. 这篇文档**肯定不包含**什么内容？

如果两个问题都能回答，且答案符合你的预期，说明文档名是合格的。

如果只能回答第1个问题，或者第2个问题答案模糊，说明文档名太宽泛，需要拆分或细化。

## 示例对比

**不合格的宽泛命名**：
- `03-注意力机制.md` → 注意力机制可以包含QKV、单头、多头、自注意力、交叉注意力、代码实现、变体...范围太广
- `04-Transformer进阶.md` → "进阶"太模糊，读者不知道讲什么

**合格的精确命名**：
- `03-注意力机制基础.md` → 明确是"基础"，限定范围
- `04-缩放点积注意力代码实现.md` → 明确是"缩放点积"的"代码实现"
- `05-多头注意力并行计算.md` → 明确是"并行计算"这一个点
