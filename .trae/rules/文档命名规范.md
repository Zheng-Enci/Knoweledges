# 文档命名规范 📝

## 核心原则

**文档主标题必须与文件名完全一致，且包含编号。**

文档名要准确反映内容边界，避免范围过广导致内容与名称不匹配。

## 具体规则

### 规则一：名称范围与内容范围精确匹配

| 错误示例 | 问题 | 正确示例 |
|---------|------|---------|
| `03-深度学习.md` | 范围太广 | `03-梯度下降优化器.md` |
| `05-注意力机制.md` | 包含太多内容 | `05-缩放点积注意力代码实现.md` |

### 规则二：名称要具体，边界清晰

好的文档名让读者一眼能看出：
- 这是什么（不是泛泛的"机制"、"模型"）
- 边界在哪里（不是包含多种东西的"综述"）

### 规则三：主标题与文件名完全一致

文件名：`04-缩放点积注意力代码实现.md`

正确主标题：
```markdown
# 04-缩放点积注意力代码实现 📚
```

错误主标题：
```markdown
# 缩放点积注意力实现 📚      <!-- 缺少编号 -->
# 04-缩放点积注意力 📚         <!-- 缺少"代码实现" -->
```

### 规则四：编号规则

- 编号从01开始递增
- 编号必须出现在文件名和主标题最前面
- 格式：`01-`、`02-`、...、`10-`

## 判断标准

看到文档名时，能否准确说出：
1. 这篇文档**肯定包含**什么？
2. 这篇文档**肯定不包含**什么？

两个问题都能回答且符合预期，说明命名合格。
