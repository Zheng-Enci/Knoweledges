# 内容要求

- 先联网搜索学习相关内容，再编写文档
- 确保内容准确性和时效性
- 添加实际案例和应用场景
- 使用表格、图表等增强可读性
- 根据实际情况调整章节结构
- **联网搜索获取内容后必须添加参考资料**：在文档中引用搜索结果时，必须在引用处（内容紧跟其后）添加所有相关的参考资料链接，有多少条就添加多少条，**紧跟引用内容**（内容写完后下一行就是参考资料），格式为：
  - 单个链接：`- [标题](URL)`
  - 多个链接：紧跟内容后列出多条，每条一行
  ```
  内容段落（联网搜索后编写的内容）...
  
  参考资料：
  - [标题1](URL1)
  - [标题2](URL2)
  - [标题3](URL3)
  ```
  - **注意**：不要放在章节末尾，要紧跟在引用内容后面
- **代码示例必须考虑跨平台兼容性**：优先选择 Windows/Linux/Mac 都支持的方案，避免使用特定平台才能运行的代码（如 Detectron2 在 Windows 上安装复杂，应选用 EfficientDet 等替代方案）
- **章节之间必须有衔接过渡**：在章节末尾或开头添加过渡段落，说明前后章节的逻辑关系、内容承接，帮助读者建立知识脉络（如"接下来我们将学习..."、"前面我们了解了...，现在深入探讨..."）
- **文档之间必须多互动联系**：
  - **每篇文档开头必须回顾上篇文档学习内容，并附上链接**（如"通过上篇文档[08c. 检索算法与策略-混合检索](https://juejin.cn/...)的学习，我们掌握了..."）
  - **文档内涉及其他文档知识点时，必须附上链接**（如"在[08a. 检索算法与策略-稠密检索技术](https://juejin.cn/...)中我们学习了..."）
  - 跨文档引用必须具体到章节，方便读者定位
  - 链接格式：掘金链接 / CSDN链接
- **文档末尾必须添加最后更新时间**：每篇文档末尾必须包含文档最后更新时间，格式为"最后更新时间：YYYY-MM-DD"
