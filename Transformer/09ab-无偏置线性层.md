## 1. 什么是无偏置线性层 📖

> 本章介绍 `nn.Linear` 的 `bias` 参数及其作用

在 PyTorch 中，`nn.Linear` 是最常用的全连接层（线性层），它执行以下运算：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
y = x \times W^T + b
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

其中：
- $x$ 是输入张量
- $W$ 是权重矩阵（可学习参数）
- $b$ 是偏置向量（可学习参数）
- $y$ 是输出张量

**什么是"无偏置线性层"？**

当创建 `nn.Linear` 时设置 `bias=False`，则运算简化为：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
y = x \times W^T
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

**去掉了偏置项 $b$**，只保留线性变换（矩阵乘法）。

```python
import torch                                              # 导入 PyTorch 核心库
import torch.nn as nn                                     # 导入神经网络模块

# 创建带偏置的线性层（默认行为）
linear_with_bias = nn.Linear(in_features=64, out_features=128, bias=True)   # 带偏置：y = xW^T + b
print(f"带偏置 - 权重形状: {linear_with_bias.weight.shape}")                # 打印权重形状：[128, 64]
print(f"带偏置 - 偏置形状: {linear_with_bias.bias.shape}")                  # 打印偏置形状：[128]

# 创建无偏置的线性层
linear_without_bias = nn.Linear(in_features=64, out_features=128, bias=False)  # 无偏置：y = xW^T
print(f"无偏置 - 权重形状: {linear_without_bias.weight.shape}")             # 打印权重形状：[128, 64]
print(f"无偏置 - 偏置属性: {linear_without_bias.bias}")                     # 打印：None（无偏置）
```

运行结果：

```
带偏置 - 权重形状: torch.Size([128, 64])
带偏置 - 偏置形状: torch.Size([128])
无偏置 - 权重形状: torch.Size([128, 64])
无偏置 - 偏置属性: None
```

> 💡 在 Transformer 的注意力机制中，**Q、K、V 投影层通常都设置 `bias=False`**，这是业界标准做法。

## 2. 为什么 Transformer 移除偏置 🤔

> 本章从数学和直觉两个角度解释为何移除偏置

### 2.1 核心原因概览

Transformer 在以下位置使用 `bias=False`：

| 层类型 | 是否用偏置 | 原因 |
|--------|-----------|------|
| **Q 投影层** | `bias=False` | Softmax 会抵消 Query 偏置 |
| **K 投影层** | `bias=False` | 数学证明：Key 偏置完全冗余 |
| **V 投影层** | `bias=False` | LayerNorm 使偏置失去作用 |
| **注意力输出投影** | `bias=False` | 同上，LayerNorm 抵消偏置 |
| **FFN 中间层** | `bias=True` | 激活函数前需要偏置 |
| **FFN 输出层** | `bias=True` | 同上 |
| **最终分类层** | `bias=True` | 输出 logits 需要偏置 |

### 2.2 直觉解释 💡

**LayerNorm 的作用**

Transformer 的每个子层（注意力层、FFN）前后都有 LayerNorm：

```python
# Transformer 标准结构
x = LayerNorm(x + Attention(x))                          # LayerNorm 会减去均值、除以标准差
x = LayerNorm(x + FFN(x))                                # 归一化消除了偏置的平移作用
```

LayerNorm 会对每个样本的特征做归一化：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\text{LayerNorm}(x) = \frac{x - \mu}{\sigma} \times \gamma + \beta
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

- $\mu$ 是均值
- $\sigma$ 是标准差
- $\gamma$ 和 $\beta$ 是可学习的缩放和平移参数

**关键点**：LayerNorm 本身就有平移参数 $\beta$，如果在 Linear 层再加偏置 $b$，两者功能重叠，造成**参数冗余**。

> 💡 这就像给同一个函数加了两次"平移"，完全没必要。

### 2.3 节省参数量

移除偏置可以节省参数量：

```python
d_model = 512                                           # 模型维度

# 带偏置的 QKV 投影
qkv_with_bias = nn.Linear(d_model, 3 * d_model, bias=True)   # 参数量：512×1536 + 1536 = 787,968
print(f"带偏置参数量: {qkv_with_bias.weight.numel() + qkv_with_bias.bias.numel()}")  # 打印总参数量

# 无偏置的 QKV 投影
qkv_without_bias = nn.Linear(d_model, 3 * d_model, bias=False)  # 参数量：512×1536 = 786,432
print(f"无偏置参数量: {qkv_without_bias.weight.numel()}")        # 打印总参数量

# 节省的参数量
saved_params = qkv_with_bias.bias.numel()                      # 节省：1536 个参数
print(f"节省参数量: {saved_params}")                           # 打印节省的参数
```

虽然单个层节省的参数不多（1536 个），但在大模型中：
- **GPT-3**（1750 亿参数）：多个层累积节省可观
- **推理加速**：减少内存访问和计算开销
- **显存优化**：更小的模型占用更少的 GPU 显存

## 3. Key 偏置的冗余性证明 📐

> 本章通过详细数学推导证明 Key 偏置在注意力机制中完全冗余

### 3.1 注意力公式回顾

缩放点积注意力的核心公式：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{Q \times K^T}{\sqrt{d_k}}\right) \times V
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

### 3.2 数学推导 🔬

假设我们给 Q 和 K 的投影都加上偏置：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
Q = X \times W_Q + b_Q
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
K = X \times W_K + b_K
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

其中：
- $X$ 是输入张量
- $W_Q$ 和 $W_K$ 是权重矩阵
- $b_Q$ 和 $b_K$ 是偏置向量

**计算注意力分数**：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\text{Scores} = Q \times K^T
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

代入 $Q$ 和 $K$：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\text{Scores} = (X \times W_Q + b_Q) \times (X \times W_K + b_K)^T
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

展开：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\text{Scores} = (X \times W_Q + b_Q) \times (W_K^T \times X^T + b_K^T)
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\text{Scores} = X \times W_Q \times W_K^T \times X^T + X \times W_Q \times b_K^T + b_Q \times W_K^T \times X^T + b_Q \times b_K^T
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

**关键观察**：

当我们对 Scores 应用 Softmax 时，**Softmax 具有平移不变性**：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\text{softmax}(x + c) = \text{softmax}(x)
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

其中 $c$ 是任意常数。

**为什么 Softmax 有平移不变性？**

Softmax 的定义：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\text{softmax}(x_i) = \frac{e^{x_i}}{\sum_{j} e^{x_j}}
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

如果给所有 $x_i$ 加上同一个常数 $c$：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\text{softmax}(x_i + c) = \frac{e^{x_i + c}}{\sum_{j} e^{x_j + c}} = \frac{e^{x_i} \times e^c}{e^c \times \sum_{j} e^{x_j}} = \frac{e^{x_i}}{\sum_{j} e^{x_j}} = \text{softmax}(x_i)
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

**结论**：分子和分母的 $e^c$ 相互抵消，结果不变！

**应用到注意力机制**：

在注意力分数中，$b_K$ 对每个 Query 位置都是**相同的常数向量**。对于第 $i$ 个 Query 位置：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\text{Scores}[i] = Q[i] \times (K[1] + b_K)^T, Q[i] \times (K[2] + b_K)^T, \ldots
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

展开后：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\text{Scores}[i, j] = Q[i] \times K[j]^T + Q[i] \times b_K^T
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

其中 $Q[i] \times b_K^T$ 对于第 $i$ 行的所有 $j$ 都是**相同的常数**。

应用 Softmax 后，这个常数被抵消：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\text{softmax}(\text{Scores}[i]) = \text{softmax}(Q[i] \times K^T + \text{常数}) = \text{softmax}(Q[i] \times K^T)
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

**最终结论**：

- **$b_K$（Key 偏置）对注意力权重完全没有影响** → 完全冗余，可以安全移除
- **$b_Q$（Query 偏置）虽然影响分数，但不影响 Softmax 后的权重分布** → 也可以移除

### 3.3 代码验证 🔍

让我们用代码验证 Key 偏置是否真的冗余：

```python
import torch.nn as nn                                     # 导入神经网络模块
import torch.nn.functional as F                           # 导入函数式 API


"""验证 Key 偏置的冗余性

参数:
    无
    
返回:
    无（打印对比结果）
    
示例:
    verify_key_bias_redundancy()
"""
def verify_key_bias_redundancy():
    torch.manual_seed(42)                                 # 设置随机种子，保证结果可复现
    
    # 创建输入张量，示例：batch_size=2, seq_len=5, d_model=64
    batch_size, seq_len, d_model = 2, 5, 64
    X = torch.randn(batch_size, seq_len, d_model)         # 随机输入 X：[2, 5, 64]
    
    # 方案1：带偏置的 QKV 投影
    W_Q = nn.Linear(d_model, d_model, bias=True)          # Query 投影带偏置
    W_K = nn.Linear(d_model, d_model, bias=True)          # Key 投影带偏置
    W_V = nn.Linear(d_model, d_model, bias=True)          # Value 投影带偏置
    
    Q_with_bias = W_Q(X)                                  # Q带偏置：[2, 5, 64]
    K_with_bias = W_K(X)                                  # K带偏置：[2, 5, 64]
    V_with_bias = W_V(X)                                  # V带偏置：[2, 5, 64]
    
    # 计算带偏置的注意力分数，数据流动：Q[2,5,64] × K^T[2,64,5] → scores[2,5,5]
    scores_with_bias = torch.matmul(Q_with_bias, K_with_bias.transpose(-2, -1))  # 注意力分数：[2, 5, 5]
    attention_with_bias = torch.softmax(scores_with_bias, dim=-1)                # 注意力权重：[2, 5, 5]
    
    # 方案2：无偏置的 QKV 投影
    W_Q_no_bias = nn.Linear(d_model, d_model, bias=False)  # Query 投影无偏置
    W_K_no_bias = nn.Linear(d_model, d_model, bias=False)  # Key 投影无偏置
    W_V_no_bias = nn.Linear(d_model, d_model, bias=False)  # Value 投影无偏置
    
    # 复制相同的权重（排除偏置）
    W_Q_no_bias.weight.data = W_Q.weight.data.clone()      # 复制 Q 权重
    W_K_no_bias.weight.data = W_K.weight.data.clone()      # 复制 K 权重
    W_V_no_bias.weight.data = W_V.weight.data.clone()      # 复制 V 权重
    
    Q_no_bias = W_Q_no_bias(X)                            # Q无偏置：[2, 5, 64]
    K_no_bias = W_K_no_bias(X)                            # K无偏置：[2, 5, 64]
    V_no_bias = W_V_no_bias(X)                            # V无偏置：[2, 5, 64]
    
    # 计算无偏置的注意力分数，数据流动：Q[2,5,64] × K^T[2,64,5] → scores[2,5,5]
    scores_no_bias = torch.matmul(Q_no_bias, K_no_bias.transpose(-2, -1))      # 注意力分数：[2, 5, 5]
    attention_no_bias = torch.softmax(scores_no_bias, dim=-1)                  # 注意力权重：[2, 5, 5]
    
    # 对比结果
    print("=" * 60)                                       # 打印分隔线
    print("Key 偏置冗余性验证")                            # 打印标题
    print("=" * 60)                                       # 打印分隔线
    print(f"带偏置的注意力权重（第一个样本，第一个头）:")     # 打印说明
    print(attention_with_bias[0])                         # 打印带偏置权重
    print(f"\n无偏置的注意力权重（第一个样本，第一个头）:")   # 打印说明
    print(attention_no_bias[0])                           # 打印无偏置权重
    
    # 计算差异
    diff = (attention_with_bias - attention_no_bias).abs().max().item()        # 计算最大绝对差异
    print(f"\n最大差异: {diff:.2e}")                      # 打印差异（科学计数法）
    
    if diff < 1e-6:                                       # 判断差异是否极小
        print("✅ Key 偏置确实冗余！权重几乎完全相同")      # 打印结论
    else:                                                 # 差异较大
        print("❌ 存在差异，需要检查")                     # 打印警告
    print("=" * 60)                                       # 打印分隔线
```

## 4. PyTorch 代码实现 💻

> 本章提供完整的 Transformer 注意力层实现，展示无偏置线性层的使用

### 4.1 标准 Transformer 注意力层实现

```python
import torch                                              # 导入 PyTorch 核心库
import torch.nn as nn                                     # 导入神经网络模块
import math                                               # 导入数学库


"""Transformer 多头注意力机制（无偏置版本）

参数:
    d_model: 模型维度（默认512）
    n_heads: 注意力头数（默认8）
    dropout: Dropout概率（默认0.1）
    
示例:
    attention = MultiHeadAttention(d_model=512, n_heads=8, dropout=0.1)
"""
class MultiHeadAttention(nn.Module):
    
    """初始化多头注意力模块
    
    参数:
        d_model: 模型维度（默认512）
        n_heads: 注意力头数（默认8）
        dropout: Dropout概率（默认0.1）
        
    返回:
        无
        
    示例:
        self.attn = MultiHeadAttention(d_model=512, n_heads=8)
    """
    def __init__(self, d_model=512, n_heads=8, dropout=0.1):
        super(MultiHeadAttention, self).__init__()
        
        # 基础参数检查
        assert d_model % n_heads == 0, "d_model 必须能被 n_heads 整除"  # 确保维度可均分
        
        self.d_model = d_model                            # 保存模型维度
        self.n_heads = n_heads                            # 保存注意力头数
        self.d_k = d_model // n_heads                     # 计算每个头的维度，示例：512/8=64
        
        # QKV 投影层（无偏置！）
        self.W_Q = nn.Linear(d_model, d_model, bias=False)   # Query 投影：[d_model, d_model]，bias=False
        self.W_K = nn.Linear(d_model, d_model, bias=False)   # Key 投影：[d_model, d_model]，bias=False
        self.W_V = nn.Linear(d_model, d_model, bias=False)   # Value 投影：[d_model, d_model]，bias=False
        
        # 输出投影层（无偏置！）
        self.W_O = nn.Linear(d_model, d_model, bias=False)   # 输出投影：[d_model, d_model]，bias=False
        
        # Dropout 层
        self.dropout = nn.Dropout(dropout)                # Dropout 层，防止过拟合
    
    """前向传播计算多头注意力
    
    参数:
        x: 输入张量 [batch_size=2, seq_len=10, d_model=512]
        mask: 可选掩码张量
        
    返回:
        output: 注意力输出 [2, 10, 512]
        attention_weights: 注意力权重 [2, 8, 10, 10]
        
    示例:
        output, weights = attention(x, mask=None)
    """
    def forward(self, x, mask=None):
        batch_size, seq_len, d_model = x.size()           # 获取输入形状，示例：[2, 10, 512]
        
        # 1. 线性投影（无偏置！），数据流动：x[2,10,512] → Q[2,10,512], K[2,10,512], V[2,10,512]
        Q = self.W_Q(x)                                   # Query 投影：[2, 10, 512]
        K = self.W_K(x)                                   # Key 投影：[2, 10, 512]
        V = self.W_V(x)                                   # Value 投影：[2, 10, 512]
        
        # 2. 分割多头，数据流动：Q[2,10,512] → Q[2,10,8,64] → Q[2,8,10,64]
        Q = Q.view(batch_size, seq_len, self.n_heads, self.d_k).transpose(1, 2)  # [2, 8, 10, 64]
        K = K.view(batch_size, seq_len, self.n_heads, self.d_k).transpose(1, 2)  # [2, 8, 10, 64]
        V = V.view(batch_size, seq_len, self.n_heads, self.d_k).transpose(1, 2)  # [2, 8, 10, 64]
        
        # 3. 计算缩放点积注意力，数据流动：Q[2,8,10,64] × K^T[2,8,64,10] → scores[2,8,10,10]
        scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(self.d_k)  # 注意力分数：[2, 8, 10, 10]
        
        # 4. 应用掩码（如有）
        if mask is not None:                              # 判断是否提供掩码
            scores = scores.masked_fill(mask == 0, float('-inf'))  # 填充 -inf，屏蔽无效位置
        
        # 5. Softmax 归一化，数据流动：scores[2,8,10,10] → attention_weights[2,8,10,10]（每行和为1）
        attention_weights = torch.softmax(scores, dim=-1)  # 注意力权重：[2, 8, 10, 10]
        attention_weights = self.dropout(attention_weights)  # 应用 Dropout
        
        # 6. 加权求和，数据流动：attention_weights[2,8,10,10] × V[2,8,10,64] → output[2,8,10,64]
        output = torch.matmul(attention_weights, V)       # 加权求和：[2, 8, 10, 64]
        
        # 7. 合并多头，数据流动：output[2,8,10,64] → output[2,10,8,64] → output[2,10,512]
        output = output.transpose(1, 2).contiguous().view(batch_size, seq_len, d_model)  # [2, 10, 512]
        
        # 8. 输出投影（无偏置！），数据流动：output[2,10,512] → output[2,10,512]
        output = self.W_O(output)                         # 输出投影：[2, 10, 512]
        
        return output, attention_weights                  # 返回输出和注意力权重


"""测试多头注意力模块

参数:
    无
    
返回:
    output: 注意力输出 [batch_size=2, seq_len=10, d_model=512]
    weights: 注意力权重 [2, 8, 10, 10]
    
示例:
    output, weights = test_multihead_attention()
"""
def test_multihead_attention():
    torch.manual_seed(42)                                 # 设置随机种子
    
    # 参数设置
    batch_size = 2                                        # 批次大小
    seq_len = 10                                          # 序列长度
    d_model = 512                                         # 模型维度
    n_heads = 8                                           # 注意力头数
    
    # 创建输入张量，示例：[2, 10, 512]
```

## 5. 何时保留偏置 🎯

> 本章介绍哪些场景应该保留偏置，哪些场景可以移除

### 5.1 应该移除偏置的场景 ✅

| 场景 | 原因 | 示例 |
|------|------|------|
| **Q/K/V 投影层** | Softmax 抵消偏置 | `nn.Linear(d_model, d_model, bias=False)` |
| **注意力输出投影** | LayerNorm 抵消偏置 | `nn.Linear(d_model, d_model, bias=False)` |
| **Embedding 后投影** | LayerNorm 在前 | `nn.Linear(embed_dim, d_model, bias=False)` |

### 5.2 应该保留偏置的场景 ⚠️

| 场景 | 原因 | 示例 |
|------|------|------|
| **FFN 中间层** | GELU/ReLU 激活前需要偏置 | `nn.Linear(d_model, d_ff, bias=True)` |
| **FFN 输出层** | 同上 | `nn.Linear(d_ff, d_model, bias=True)` |
| **最终分类层** | 输出 logits 需要偏置 | `nn.Linear(d_model, num_classes, bias=True)` |
| **没有 LayerNorm 的层** | 无归一化抵消 | 任何独立的 Linear 层 |

### 5.3 代码示例：完整的 Transformer 编码器层

```python
import torch                                              # 导入 PyTorch 核心库
import torch.nn as nn                                     # 导入神经网络模块


"""完整的 Transformer 编码器层（偏置使用规范）

参数:
    d_model: 模型维度（默认512）
    n_heads: 注意力头数（默认8）
    d_ff: FFN隐藏层维度（默认2048）
    dropout: Dropout概率（默认0.1）
    
示例:
    encoder_layer = TransformerEncoderLayer(d_model=512, n_heads=8, d_ff=2048)
"""
class TransformerEncoderLayer(nn.Module):
    
    """初始化编码器层
    
    参数:
        d_model: 模型维度（默认512）
        n_heads: 注意力头数（默认8）
        d_ff: FFN隐藏层维度（默认2048）
        dropout: Dropout概率（默认0.1）
        
    返回:
        无
        
    示例:
        self.encoder = TransformerEncoderLayer(d_model=512, n_heads=8)
    """
    def __init__(self, d_model=512, n_heads=8, d_ff=2048, dropout=0.1):
        
        # 多头注意力层（无偏置！）
        self.self_attn = nn.MultiheadAttention(           # PyTorch 原生多头注意力
            embed_dim=d_model,                            # 模型维度：512
            num_heads=n_heads,                            # 注意力头数：8
            dropout=dropout,                              # Dropout 概率：0.1
            bias=False,                                   # QKV 无偏置！
            batch_first=True                              # 批次优先：[batch, seq, d_model]
        )
        
        # 前馈神经网络 FFN（有偏置！）
        self.ffn = nn.Sequential(                         # FFN 包含两层 Linear
            nn.Linear(d_model, d_ff, bias=True),          # 第一层：[512, 2048]，bias=True ✅
            nn.GELU(),                                    # GELU 激活函数
            nn.Dropout(dropout),                          # Dropout
            nn.Linear(d_ff, d_model, bias=True)           # 第二层：[2048, 512]，bias=True ✅
        )
        
        # LayerNorm 层
        self.norm1 = nn.LayerNorm(d_model)                # 注意力后的 LayerNorm
        self.norm2 = nn.LayerNorm(d_model)                # FFN 后的 LayerNorm
        
        # Dropout 层
        self.dropout1 = nn.Dropout(dropout)               # 注意力后的 Dropout
        self.dropout2 = nn.Dropout(dropout)               # FFN 后的 Dropout
    
    """前向传播
    
    参数:
        x: 输入张量 [batch_size=2, seq_len=10, d_model=512]
        mask: 可选掩码
        
    返回:
        output: 编码器输出 [2, 10, 512]
        
    示例:
        output = encoder_layer(x, mask=None)
    """
    def forward(self, x, mask=None):
        # 1. 自注意力 + 残差连接 + LayerNorm
        # 数据流动：x[2,10,512] → attn_output[2,10,512] → x+attn[2,10,512] → norm[2,10,512]
        attn_output, _ = self.self_attn(x, x, x, attn_mask=mask)  # 自注意力：[2, 10, 512]
        x = self.norm1(x + self.dropout1(attn_output))    # 残差 + Dropout + LayerNorm
        
        # 2. FFN + 残差连接 + LayerNorm
        # 数据流动：x[2,10,512] → ffn_output[2,10,512] → x+ffn[2,10,512] → norm[2,10,512]
        ffn_output = self.ffn(x)                          # FFN：[2, 10, 512]
        x = self.norm2(x + self.dropout2(ffn_output))     # 残差 + Dropout + LayerNorm
        
        return x                                          # 返回编码器输出


"""测试编码器层

参数:
    无
    
返回:
    output: 编码器输出 [batch_size=2, seq_len=10, d_model=512]
    
示例:
    output = test_encoder_layer()
"""
def test_encoder_layer():
    torch.manual_seed(42)                                 # 设置随机种子
    
    # 参数设置
    batch_size = 2                                        # 批次大小
    seq_len = 10                                          # 序列长度
    d_model = 512                                         # 模型维度
    n_heads = 8                                           # 注意力头数
    d_ff = 2048                                           # FFN 隐藏层维度
    
    # 创建输入
    x = torch.randn(batch_size, seq_len, d_model)         # 随机输入：[2, 10, 512]
    
    # 创建编码器层
    encoder = TransformerEncoderLayer(                    # 创建编码器层
        d_model=d_model,                                  # 模型维度：512
        n_heads=n_heads,                                  # 头数：8
        d_ff=d_ff,                                        # FFN 维度：2048
        dropout=0.0                                       # Dropout：0.0（测试用）
    )
    
    # 前向传播，数据流动：x[2,10,512] → output[2,10,512]
    output = encoder(x)
    
    print("=" * 60)                                       # 打印分隔线
    print("Transformer 编码器层测试")                      # 打印标题
    print("=" * 60)                                       # 打印分隔线
    print(f"输入形状: {x.shape}")                         # 打印输入形状
    print(f"输出形状: {output.shape}")                    # 打印输出形状
    
    # 检查偏置使用情况
    print(f"\n偏置使用检查:")                             # 打印标题
    print(f"  注意力 QKV 偏置: {encoder.self_attn.in_proj_bias}")  # 应为 None
    print(f"  注意力输出偏置: {encoder.self_attn.out_proj.bias}")  # 应为 None
    print(f"  FFN 第一层偏置: {encoder.ffn[0].bias is not None}")  # 应为 True ✅
    print(f"  FFN 第二层偏置: {encoder.ffn[3].bias is not None}")  # 应为 True ✅
    
    # 参数量统计
    total_params = sum(p.numel() for p in encoder.parameters())    # 计算总参数量
    print(f"\n总参数量: {total_params:,}")                # 打印总参数量
    print("=" * 60)                                       # 打印分隔线
    
    return output                                         # 返回输出


if __name__ == "__main__":
    test_encoder_layer()                                  # 运行测试
```

## 6. 总结 📝

> 本章回顾核心要点

本节我们详解了 Transformer 中无偏置线性层的使用，核心要点回顾：

| 要点 | 说明 |
|------|------|
| **Q/K/V 投影层** | `bias=False`，Softmax 使 Key 偏置冗余 |
| **注意力输出投影** | `bias=False`，LayerNorm 抵消偏置 |
| **FFN 层** | `bias=True`，激活函数前需要偏置 |
| **最终分类层** | `bias=True`，输出 logits 需要偏置 |
| **参数节省** | 单层节省约 0.19%，大模型累积效果显著 |

🔴 **关键理解**：

1. **Key 偏置冗余性**：数学证明 + 代码验证，Softmax 的平移不变性使 Key 偏置完全无效
2. **LayerNorm 的作用**：归一化层的平移参数 $\beta$ 与 Linear 偏置功能重叠
3. **工程优化**：移除不必要的偏置可节省参数量和内存访问开销
4. **不是所有层都该移除偏置**：FFN 和分类层仍需保留偏置

**最佳实践**：

```python
# ✅ 推荐做法
self.q = nn.Linear(d_model, d_model, bias=False)      # Q 无偏置
self.k = nn.Linear(d_model, d_model, bias=False)      # K 无偏置
self.v = nn.Linear(d_model, d_model, bias=False)      # V 无偏置
self.out = nn.Linear(d_model, d_model, bias=False)    # 输出无偏置

self.ffn1 = nn.Linear(d_model, d_ff, bias=True)       # FFN 有偏置 ✅
self.ffn2 = nn.Linear(d_ff, d_model, bias=True)       # FFN 有偏置 ✅

# ❌ 不推荐
self.q = nn.Linear(d_model, d_model, bias=True)       # Q 偏置冗余
self.k = nn.Linear(d_model, d_model, bias=True)       # K 偏置完全无效
```
