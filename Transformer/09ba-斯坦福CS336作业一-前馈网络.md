## 1. 前馈网络核心概念 📚

> 本章介绍 FFN 的基本结构、数学公式及其在 Transformer 中的定位

在 Transformer 架构中，前馈网络（Feed-Forward Network, FFN）是每一个 Transformer Block 的核心组件之一，位于自注意力层（Self-Attention）之后。

### 1.1 核心公式 📝

前馈网络的数学表达式为：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\text{FFN}(x) = \text{Activation}(x \cdot W_1 + b_1) \cdot W_2 + b_2
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

其中：
- $x \in \mathbb{R}^{d_{\text{model}}}$：输入向量（来自注意力层的输出）
- $W_1 \in \mathbb{R}^{d_{\text{ff}} \times d_{\text{model}}}$：第一层权重矩阵（升维）
- $b_1 \in \mathbb{R}^{d_{\text{ff}}}$：第一层偏置向量
- $\text{Activation}$：非线性激活函数（如 ReLU、GELU、SwiGLU）
- $W_2 \in \mathbb{R}^{d_{\text{model}} \times d_{\text{ff}}}$：第二层权重矩阵（降维）
- $b_2 \in \mathbb{R}^{d_{\text{model}}}$：第二层偏置向量

**维度变化流程**：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\mathbb{R}^{d_{\text{model}}} \xrightarrow{W_1} \mathbb{R}^{d_{\text{ff}}} \xrightarrow{\text{Activation}} \mathbb{R}^{d_{\text{ff}}} \xrightarrow{W_2} \mathbb{R}^{d_{\text{model}}}
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

以原始 Transformer 论文为例：
- $d_{\text{model}} = 512$（模型维度）
- $d_{\text{ff}} = 2048$（前馈网络中间维度，通常是 $d_{\text{model}}$ 的 4 倍）

### 1.2 FFN 在 Transformer 中的定位 🎯

Self-Attention 负责让不同 Token 之间"互通信息"，而 FFN 负责让每个 Token "独立思考"，做非线性的深度加工。

**关键特性**：

1. **逐位置处理（Position-Wise）**：FFN 独立处理每个位置的 Token，不跨位置共享信息
2. **维度先升后降**：先扩展到高维空间进行特征提取，再压缩回原维度
3. **保持输入输出形状一致**：输入和输出都是 $d_{\text{model}}$ 维度，方便残差连接

在斯坦福 CS336 作业一中，你需要从零实现这个组件，作为构建完整 Transformer 语言模型的关键步骤。

## 2. FFN 的三大核心作用 🔑

> 本章深入理解 FFN 在 Transformer 中不可替代的作用：非线性变换、知识存储、特征提取

一个自然的问题是：**注意力机制已经能够捕捉全局依赖了，为什么还需要 FFN？**

我们可以从三个核心作用来回答这个问题。

### 2.1 作用一：引入非线性变换（数学必要性）

**如果没有 FFN，多层 Transformer 会退化为线性模型。**

假设 FFN 中没有激活函数（即 $\text{Activation}$ 是恒等函数），那么：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\text{FFN}(x) = (x \cdot W_1 + b_1) \cdot W_2 + b_2 = x \cdot (W_1 W_2) + (b_1 W_2 + b_2)
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

令 $W' = W_1 W_2$ 和 $b' = b_1 W_2 + b_2$，则：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\text{FFN}(x) = x \cdot W' + b'
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

这只是一个简单的线性变换！**两层线性层叠加等价于一层线性层**，模型失去了学习复杂特征组合的能力。

**自注意力的局限性**：

自注意力计算的是值向量（Value）的加权平均：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

尽管注意力权重通过 Softmax 引入了非线性，但**对 Value 的操作本质上仍然是线性组合**。无论堆叠多少层注意力，无论多么精心地调整 Query 和 Key 的投影，从输入到输出的映射在 Value 上始终是线性的。

**FFN 提供了缺失的非线性**：

FFN 通过"升维 → 激活 → 降维"的三步操作，引入了真正的非线性变换，使模型能够学习复杂的、高维的决策边界。根据通用近似定理（Universal Approximation Theorem），带有非线性的网络可以近似任意连续函数。

### 2.2 作用二：存储事实知识（模型的"记忆层"）

**FFN 是 Transformer 中事实知识的主要载体。**

研究表明（Geva et al., 2021），FFN 层实际上承担了**键值记忆（Key-Value Memory）**的功能：

1. **第一层（$W_1$）充当"键"**：每一列对应一种激活模式，匹配输入中的特定模式
2. **第二层（$W_2$）充当"值"**：每一行存储与激活模式关联的知识内容

**工作原理**：

- 当输入包含"埃菲尔铁塔"时，$W_1$ 的某些列会被强烈激活（识别出"地标"模式）
- 这些激活通过非线性函数后，$W_2$ 的对应行会被触发，输出"在巴黎"等相关知识
- 这种机制使模型能够"回忆"训练中学到的事实

**实验证据**：

研究者通过编辑 FFN 中的特定参数，能够精确地修改模型存储的事实知识：
- 将"埃菲尔铁塔在巴黎"→"埃菲尔铁塔在伦敦"
- 修改后，模型对相关问题的回答会发生预期改变

这证明 FFN 不仅仅是特征变换器，更是**分布式事实数据库**。在 GPT 等模型中，FFN 层持有约 67% 的模型参数，这些参数编码了大量的世界知识。

### 2.3 作用三：独立特征提取（让每个 Token "独立思考"）

**Attention 负责"横向交流"，FFN 负责"纵向深化"。**

Transformer 的设计遵循优雅的分工原则：

| 组件 | 职责 | 信息流向 | 类比 |
|------|------|---------|------|
| **Self-Attention** | 跨位置信息路由 | 位置 ↔ 位置（横向） | "开会交流"：每个 Token 环顾四周，收集上下文 |
| **FFN** | 逐位置特征变换 | 通道 ↔ 通道（纵向） | "闭门思考"：每个 Token 独立加工收集到的信息 |

**具体流程**：

1. **Attention 阶段**：
   - 每个 Token 通过注意力机制"看到"句子中的其他 Token
   - 根据相关性分配注意力权重，聚合上下文信息
   - 输出：融合了全文上下文的 Token 表示

2. **FFN 阶段**：
   - 每个 Token **不再看其他位置**，只处理自己已经满载上下文信息的向量
   - 通过非线性变换和通道混合，进行深度的内部加工
   - 输出：经过知识检索和特征深化的新表示

**为什么需要"逐位置"处理？**

- **平移不变性**：理解"the cat"的语法模式，无论它出现在句首、句中还是句末，都适用相同的变换规则
- **参数共享**：所有位置使用同一套 FFN 权重，大幅减少模型大小（不需要为每个位置学习独立的网络）
- **并行计算**：各位置独立处理，可以完全并行化，提升训练效率

**直观类比**：

想象你在写一篇关于"猫"的文章：
1. **Attention**：你查阅资料，看到"猫"与"小猫"、"动物"、"宠物"等词相关
2. **FFN**：你根据这些上下文，结合大脑中已有的知识（猫是哺乳动物、有毛、会叫等），写出更深入的内容

Attention 帮你**收集信息**，FFN 帮你**消化信息并产出新知**。

### 2.4 三大作用总结

| 作用 | 核心价值 | 如果缺失会怎样 |
|------|---------|---------------|
| **非线性变换** | 使模型能学习复杂函数 | 多层 Transformer 退化为单层线性模型 |
| **知识存储** | 编码事实知识和世界知识 | 模型无法回答事实性问题（如"巴黎是法国的首都"） |
| **特征提取** | 独立深化每个 Token 的表示 | 模型只能"搬运"信息，无法"理解"信息 |

## 3. 手动实现前馈网络 💻

> 本章从零编写 FFN 的完整代码，逐行讲解

### 3.1 完整代码实现

下面是基于 PyTorch 的完整手动实现：

```python
import torch                                              # 导入 PyTorch 核心库，提供张量运算和神经网络模块
import torch.nn as nn                                     # 导入神经网络模块，包含 Linear、Dropout 等层
import torch.nn.functional as F                           # 导入函数式 API 模块，包含 GELU、SiLU 等激活函数


"""前馈神经网络（Feed-Forward Network）的 PyTorch 实现

参数:
    d_model: 输入和输出的维度（默认512），示例：512
    d_ff: 中间隐藏层维度（默认2048），通常是 d_model 的 4 倍
    dropout: Dropout概率，用于防止过拟合（默认0.1）
    activation: 激活函数类型（默认'relu'），可选 'relu'、'gelu'、'silu'
    
示例:
    ffn = FeedForwardNetwork(d_model=512, d_ff=2048, dropout=0.1, activation='gelu')
"""
class FeedForwardNetwork(nn.Module):
    """初始化前馈网络层
    
    参数:
        d_model: 输入/输出维度（默认512），示例：512
        d_ff: 中间层维度（默认2048），示例：2048
        dropout: Dropout概率（默认0.1），示例：0.1 表示训练时随机丢弃 10% 的权重
        activation: 激活函数类型（默认'relu'），示例：'gelu'
        
    返回:
        无
        
    示例:
        self.ffn = FeedForwardNetwork(d_model=512, d_ff=2048)
    """
    def __init__(self, d_model=512, d_ff=2048, dropout=0.1, activation='relu'):
        super(FeedForwardNetwork, self).__init__()
        
        # 第一层线性变换：升维，数据流动：x[batch, seq_len, 512] → hidden[batch, seq_len, 2048]
        self.linear_1 = nn.Linear(d_model, d_ff)
        
        # Dropout 层：训练时随机丢弃部分权重，防止过拟合
        self.dropout = nn.Dropout(dropout)
        
        # 第二层线性变换：降维，数据流动：hidden[batch, seq_len, 2048] → output[batch, seq_len, 512]
        self.linear_2 = nn.Linear(d_ff, d_model)
        
        # 选择激活函数
        self.activation = activation
    
    """前向传播计算前馈网络
    
    参数:
        x: 输入张量 [batch_size, seq_len, d_model]
        
    返回:
        output: 输出张量 [batch_size, seq_len, d_model]
        
    示例:
        output = ffn(x)  # x.shape=[2, 10, 512] → output.shape=[2, 10, 512]
    """
    def forward(self, x):
        # 1. 第一层线性变换（升维）
        # 数据流动：x[batch=2, seq_len=10, d_model=512] → hidden[2, 10, d_ff=2048]
        hidden = self.linear_1(x)
        
        # 2. 应用激活函数（引入非线性）
        # 数据流动：hidden[2, 10, 2048] → activated[2, 10, 2048]（形状不变，数值分布改变）
        if self.activation == 'relu':                                         # 判断是否使用 ReLU 激活函数
            hidden = F.relu(hidden)                                           # ReLU：负数置0，正数保持不变
        elif self.activation == 'gelu':                                       # 判断是否使用 GELU 激活函数
            hidden = F.gelu(hidden)                                           # GELU：平滑的非线性变换
        elif self.activation == 'silu':                                       # 判断是否使用 SiLU/Swish 激活函数
            hidden = F.silu(hidden)                                           # SiLU：x * sigmoid(x)
        
        # 3. 应用 Dropout（训练时随机丢弃部分权重）
        # 数据流动：activated[2, 10, 2048] → dropped[2, 10, 2048]（部分变为0）
        hidden = self.dropout(hidden)
        
        # 4. 第二层线性变换（降维）
        # 数据流动：hidden[2, 10, 2048] → output[2, 10, 512]
        output = self.linear_2(hidden)
        
        return output
```

### 3.2 代码逐行解析 🔍

**第1步：第一层线性变换（升维）**

```python
hidden = self.linear_1(x)
```

- 输入形状：`[batch_size, seq_len, d_model]`，例如 `[2, 10, 512]`
- 权重矩阵 $W_1$ 形状：`[d_model, d_ff]` = `[512, 2048]`
- 输出形状：`[batch_size, seq_len, d_ff]` = `[2, 10, 2048]`

这一步将每个 Token 从 512 维扩展到 2048 维，为后续的非线性变换提供足够的表达空间。

**第2步：激活函数（引入非线性）**

```python
if self.activation == 'relu':
    hidden = F.relu(hidden)
```

激活函数是 FFN 的核心。以 ReLU 为例：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\text{ReLU}(x) = \max(0, x)
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

- 形状不变：`[2, 10, 2048]` → `[2, 10, 2048]`
- 数值变化：所有负数变为 0，正数保持不变
- 作用：引入非线性，使模型能够学习复杂的特征组合

**第3步：Dropout**

```python
hidden = self.dropout(hidden)
```

训练时随机将一部分权重置为 0（默认 10%），防止模型过度依赖某些神经元，提升泛化能力。

**第4步：第二层线性变换（降维）**

```python
output = self.linear_2(hidden)
```

- 输入形状：`[2, 10, 2048]`
- 权重矩阵 $W_2$ 形状：`[d_ff, d_model]` = `[2048, 512]`
- 输出形状：`[2, 10, 512]`

将高维特征压缩回原始维度，方便后续的残差连接。

## 4. 激活函数对比 🔬

> 本章详细对比 ReLU、GELU、SwiGLU 三种主流激活函数

### 4.1 ReLU（Rectified Linear Unit）

**公式**：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\text{ReLU}(x) = \max(0, x)
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

**特点**：
- **原始 Transformer 使用**：Vaswani et al. (2017) 在 "Attention Is All You Need" 中选择 ReLU
- **简单高效**：计算成本极低，只需一个比较操作
- **非饱和正区域**：对于 $x > 0$，导数恒为 1，梯度不会消失
- **缺点**：存在"神经元死亡"问题——负数梯度为 0，某些神经元可能永远不会被激活

**适用场景**：计算资源受限、需要快速训练的场景

```python
import torch.nn.functional as F                           # 导入函数式 API 模块

# ReLU 实现示例
x = torch.tensor([-2.0, -1.0, 0.0, 1.0, 2.0])           # 输入张量：[-2, -1, 0, 1, 2]
output = F.relu(x)                                        # ReLU 输出：[0, 0, 0, 1, 2]
```

### 4.2 GELU（Gaussian Error Linear Unit）

**公式**：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\text{GELU}(x) = x \cdot \Phi(x) = x \cdot \frac{1}{2} \left[1 + \text{erf}\left(\frac{x}{\sqrt{2}}\right)\right]
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

其中 $\Phi(x)$ 是标准正态分布的累积分布函数（CDF）。

**特点**：
- **BERT 使用**：Devlin et al. (2019) 在 BERT 中选择 GELU
- **平滑非线性**：与 ReLU 的硬截断不同，GELU 是平滑曲线，对负数也有非零输出
- **概率加权**：根据输入值在正态分布中的概率进行加权，表现更稳定
- **计算成本较高**：涉及误差函数（erf）的计算

**直观理解**：GELU 像是"软版本的 ReLU"——对于正数，输出接近 $x$；对于负数，输出接近 0 但不会完全截断。

```python
# GELU 实现示例
x = torch.tensor([-2.0, -1.0, 0.0, 1.0, 2.0])           # 输入张量：[-2, -1, 0, 1, 2]
output = F.gelu(x)                                        # GELU 输出：[-0.05, -0.16, 0, 0.84, 1.96]
```

### 4.3 SwiGLU（Swish Gated Linear Unit）

**公式**：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\text{SwiGLU}(x) = \text{SiLU}(x \cdot W_g) \otimes (x \cdot W_v)
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

其中：
- $W_g$ 和 $W_v$ 是两个不同的线性投影
- $\text{SiLU}(x) = x \cdot \sigma(x)$（Swish 激活函数，$\sigma$ 是 Sigmoid）
- $\otimes$ 表示逐元素相乘

**特点**：
- **现代 LLM 使用**：Llama、PaLM 等前沿模型都采用 SwiGLU
- **门控机制**：通过两个分支的逐元素相乘，实现动态的特征路由
- **参数量增加**：需要三个线性层（gate、value、output），但性能提升显著
- **训练更稳定**：平滑的梯度流动，适合大规模训练

**为什么 SwiGLU 表现更好？**

SwiGLU 不仅是一个激活函数，更是一个**结构化的门控模块**：
1. **Gate 分支**：学习"哪些特征应该被激活"
2. **Value 分支**：学习"特征的实际值"
3. **逐元素相乘**：动态地让重要特征通过，抑制无关特征

```python
import torch                                              # 导入 PyTorch 核心库
import torch.nn as nn                                     # 导入神经网络模块
import torch.nn.functional as F                           # 导入函数式 API


"""SwiGLU 激活函数的 PyTorch 实现

参数:
    in_features: 输入维度（默认512）
    hidden_features: 隐藏层维度（默认1365）
    
示例:
    swiglu = SwiGLU(in_features=512, hidden_features=1365)
"""
class SwiGLU(nn.Module):
    """初始化 SwiGLU 模块
    
    参数:
        in_features: 输入维度（默认512），示例：512
        hidden_features: 隐藏层维度（默认1365），通常是 in_features 的 2.6 倍
        
    返回:
        无
        
    示例:
        self.swiglu = SwiGLU(in_features=512, hidden_features=1365)
    """
    def __init__(self, in_features=512, hidden_features=1365):
        super(SwiGLU, self).__init__()
        
        # Gate 分支：学习哪些特征应该被激活
        self.gate_proj = nn.Linear(in_features, hidden_features)
        
        # Value 分支：学习特征的实际值
        self.value_proj = nn.Linear(in_features, hidden_features)
        
        # Output 投影：将高维特征压缩回原维度
        self.out_proj = nn.Linear(hidden_features, in_features)
    
    """前向传播计算 SwiGLU
    
    参数:
        x: 输入张量 [batch_size, seq_len, in_features]
        
    返回:
        output: 输出张量 [batch_size, seq_len, in_features]
        
    示例:
        output = swiglu(x)  # x.shape=[2, 10, 512] → output.shape=[2, 10, 512]
    """
    def forward(self, x):
        # 1. Gate 分支：应用 SiLU 激活函数
        # 数据流动：x[2, 10, 512] → gate[2, 10, 1365] → silu_gate[2, 10, 1365]
        gate = F.silu(self.gate_proj(x))
        
        # 2. Value 分支：线性投影（不加激活）
        # 数据流动：x[2, 10, 512] → value[2, 10, 1365]
        value = self.value_proj(x)
        
        # 3. 逐元素相乘：门控机制
        # 数据流动：gate[2, 10, 1365] ⊗ value[2, 10, 1365] → hidden[2, 10, 1365]
        hidden = gate * value
        
        # 4. Output 投影：降维
        # 数据流动：hidden[2, 10, 1365] → output[2, 10, 512]
        output = self.out_proj(hidden)
        
        return output
```

### 4.4 三种激活函数对比总结

| 特性 | ReLU | GELU | SwiGLU |
|------|------|------|--------|
| **公式** | $\max(0, x)$ | $x \cdot \Phi(x)$ | $\text{SiLU}(x W_g) \otimes (x W_v)$ |
| **计算成本** | 极低 | 中等 | 较高（3个线性层） |
| **平滑性** | 不平滑（硬截断） | 平滑 | 平滑 |
| **参数量** | 标准 | 标准 | 增加约 1.5 倍 |
| **使用模型** | 原始 Transformer | BERT | Llama、PaLM |
| **训练稳定性** | 一般 | 好 | 优秀 |
| **性能表现** | 基线 | +2~5% | +5~10% |

> 💡 **建议**：学习阶段用 ReLU 理解原理，实际项目中优先选择 GELU 或 SwiGLU 获得最佳性能。

## 5. 参数量与计算分析 📊

> 本章详细分析 FFN 的参数量占比及其在模型中的重要性

### 5.1 参数量计算

在标准 Transformer 中（$d_{\text{model}} = 512$，$d_{\text{ff}} = 2048$）：

**FFN 参数量**：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\begin{aligned}
\text{Param}_{\text{FFN}} &= d_{\text{model}} \times d_{\text{ff}} + d_{\text{ff}} \times d_{\text{model}} \\
&= 2 \times d_{\text{model}} \times d_{\text{ff}} \\
&= 2 \times 512 \times 2048 \\
&= 2,097,152
\end{aligned}
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

**Self-Attention 参数量**：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\begin{aligned}
\text{Param}_{\text{Attention}} &= 4 \times d_{\text{model}}^2 \\
&= 4 \times 512^2 \\
&= 1,048,576
\end{aligned}
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

**对比**：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\frac{\text{Param}_{\text{FFN}}}{\text{Param}_{\text{Attention}}} = \frac{2,097,152}{1,048,576} = 2
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

**结论**：FFN 的参数量是 Self-Attention 的 **2 倍**，占整个 Transformer Block 总参数量的约 **2/3**。

### 5.2 为什么 FFN 占大头？

1. **知识存储**：研究表明，FFN 层实际上承担了"记忆"的功能，存储了模型学习到的事实知识
2. **特征提取**：高维中间层（$d_{\text{ff}}$）提供了足够的表达空间，让激活函数能够提取复杂的特征组合
3. **计算瓶颈**：在推理时，FFN 往往占总计算量的 60%~70%，是优化显存和速度的关键

### 5.3 为什么 $d_{\text{ff}} = 4 \times d_{\text{model}}$？

**从表达能力角度**：

非线性函数需要在"足够宽"的空间里发挥作用。如果中间层太窄，激活函数能捕捉的特征组合就很有限；太宽又会让参数量暴涨、训练变慢。实验证明，4 倍既有足够的表达空间，又不会太贵。

**从计算平衡角度**：

当 $d_{\text{ff}} = 4 \times d_{\text{model}}$ 时，FFN 和 Attention 的计算量大致相当，整个 Transformer Block 的资源分配更加均衡。

**现代变体**：

- **Llama 使用 2.67 倍**：配合 SwiGLU 后，参数量增加，因此降低了倍数（$d_{\text{ff}} \approx 2.67 \times d_{\text{model}}$）
- **GPT-3 使用 4 倍**：保持原始比例

## 6. 完整可运行示例 🎯

> 本章提供一个从头到尾可运行的完整代码，包含三种激活函数的对比测试

```python
import torch                                              # 导入 PyTorch 核心库
import torch.nn as nn                                     # 导入神经网络模块
import torch.nn.functional as F                           # 导入函数式 API


"""前馈神经网络（Feed-Forward Network）的 PyTorch 实现

参数:
    d_model: 输入和输出的维度（默认512）
    d_ff: 中间隐藏层维度（默认2048），通常是 d_model 的 4 倍
    dropout: Dropout概率（默认0.1）
    activation: 激活函数类型（默认'relu'），可选 'relu'、'gelu'、'silu'
    
示例:
    ffn = FeedForwardNetwork(d_model=512, d_ff=2048, dropout=0.1, activation='gelu')
"""
class FeedForwardNetwork(nn.Module):
    """初始化前馈网络层
    
    参数:
        d_model: 输入/输出维度（默认512）
        d_ff: 中间层维度（默认2048）
        dropout: Dropout概率（默认0.1）
        activation: 激活函数类型（默认'relu'）
        
    返回:
        无
        
    示例:
        self.ffn = FeedForwardNetwork(d_model=512, d_ff=2048)
    """
    def __init__(self, d_model=512, d_ff=2048, dropout=0.1, activation='relu'):
        super(FeedForwardNetwork, self).__init__()
        self.linear_1 = nn.Linear(d_model, d_ff)
        self.dropout = nn.Dropout(dropout)
        self.linear_2 = nn.Linear(d_ff, d_model)
        self.activation = activation
    
    """前向传播计算前馈网络
    
    参数:
        x: 输入张量 [batch_size, seq_len, d_model]
        
    返回:
        output: 输出张量 [batch_size, seq_len, d_model]
        
    示例:
        output = ffn(x)  # x.shape=[2, 10, 512] → output.shape=[2, 10, 512]
    """
    def forward(self, x):
        # 1. 升维，数据流动：x[2, 10, 512] → hidden[2, 10, 2048]
        hidden = self.linear_1(x)
        
        # 2. 激活函数，数据流动：hidden[2, 10, 2048] → activated[2, 10, 2048]
        if self.activation == 'relu':
            hidden = F.relu(hidden)
        elif self.activation == 'gelu':
            hidden = F.gelu(hidden)
        elif self.activation == 'silu':
            hidden = F.silu(hidden)
        
        # 3. Dropout，数据流动：activated[2, 10, 2048] → dropped[2, 10, 2048]
        hidden = self.dropout(hidden)
        
        # 4. 降维，数据流动：hidden[2, 10, 2048] → output[2, 10, 512]
        output = self.linear_2(hidden)
        
        return output


"""SwiGLU 前馈网络的 PyTorch 实现

参数:
    d_model: 输入和输出的维度（默认512）
    d_ff: 中间隐藏层维度（默认1365），通常是 d_model 的 2.67 倍
    dropout: Dropout概率（默认0.1）
    
示例:
    swiglu_ffn = SwiGLUFeedForward(d_model=512, d_ff=1365, dropout=0.1)
"""
class SwiGLUFeedForward(nn.Module):
    """初始化 SwiGLU 前馈网络层
    
    参数:
        d_model: 输入/输出维度（默认512）
        d_ff: 中间层维度（默认1365）
        dropout: Dropout概率（默认0.1）
        
    返回:
        无
        
    示例:
        self.swiglu_ffn = SwiGLUFeedForward(d_model=512, d_ff=1365)
    """
    def __init__(self, d_model=512, d_ff=1365, dropout=0.1):
        super(SwiGLUFeedForward, self).__init__()
        
        # Gate、Value、Output 三个线性层
        self.gate_proj = nn.Linear(d_model, d_ff)
        self.value_proj = nn.Linear(d_model, d_ff)
        self.out_proj = nn.Linear(d_ff, d_model)
        self.dropout = nn.Dropout(dropout)
    
    """前向传播计算 SwiGLU 前馈网络
    
    参数:
        x: 输入张量 [batch_size, seq_len, d_model]
        
    返回:
        output: 输出张量 [batch_size, seq_len, d_model]
        
    示例:
        output = swiglu_ffn(x)  # x.shape=[2, 10, 512] → output.shape=[2, 10, 512]
    """
    def forward(self, x):
        # 1. Gate 和 Value 分支
        gate = F.silu(self.gate_proj(x))                                    # Gate：SiLU 激活
        value = self.value_proj(x)                                          # Value：线性投影
        
        # 2. 逐元素相乘（门控机制）
        hidden = gate * value
        
        # 3. Dropout 和 Output 投影
        hidden = self.dropout(hidden)
        output = self.out_proj(hidden)
        
        return output


"""测试前馈网络

参数:
    无
    
返回:

"""
def test_feedforward():
    # 设置随机种子，保证结果可复现
    torch.manual_seed(42)
    
    # 参数设置
    batch_size = 2
    seq_len = 10
    d_model = 512
    d_ff = 2048
```

## 7. 总结 📝

本节我们完成了斯坦福 CS336 作业一中的前馈网络实现，核心要点回顾：

| 组件 | 操作 | 代码对应 |
|------|------|---------|
| 升维 | $x \cdot W_1 + b_1$ | `self.linear_1(x)` |
| 激活 | $\text{Activation}(\cdot)$ | `F.relu/gelu/silu(hidden)` |
| Dropout | 随机丢弃 | `self.dropout(hidden)` |
| 降维 | $\text{hidden} \cdot W_2 + b_2$ | `self.linear_2(hidden)` |

🔴 **关键理解**：

1. **FFN 的作用**：让每个 Token 独立进行非线性特征提取，与 Attention 的"横向交流"互补
2. **为什么需要非线性**：没有激活函数的多层线性变换等价于单层，失去表达能力
3. **激活函数选择**：
   - 学习阶段：ReLU（简单直观）
   - 实际项目：GELU（BERT 标准）或 SwiGLU（Llama 标准，性能最优）
4. **参数量占比**：FFN 占 Transformer Block 总参数量的约 2/3，是模型"记忆"的主要载体
5. **维度设计**：$d_{\text{ff}} = 4 \times d_{\text{model}}$ 是经典选择，SwiGLU 可降至 2.67 倍

在斯坦福 CS336 作业一中，你需要将这些组件与 Self-Attention、LayerNorm、残差连接等组合，构建完整的 Transformer Block，最终训练出一个从零开始的语言模型。
