## 1. 中值滤波是什么 🤔

> 📦 本章介绍中值滤波的基本概念、定义和典型应用场景

### 1.1 基本定义 📝

**中值滤波（Median Filter）** 是一种**非线性数字滤波技术**，广泛应用于图像处理、信号处理和数据去噪领域。它的核心思想非常简单却极其有效：

> 用一个滑动窗口覆盖数据点，将窗口内所有值**排序后取中位数**，用这个中位数替换中心位置的原始值。

**直观类比** 🎯：想象你在一个班级里测量身高，每个人的身高可能因为测量误差有偏差。中值滤波就像是让每个人站在一群同学中间，然后取这群人"最中间"的身高作为代表值——这样可以排除特别高或特别矮的极端值影响。

### 1.2 为什么叫"中值"？🤔

**中值（Median）** 是统计学中的一个概念，表示一组数据排序后位于**正中间**的那个值。

举个例子 🌰：
- 数据：`[3, 7, 1, 9, 5]`
- 排序后：`[1, 3, 5, 7, 9]`
- **中值 = 5**（正中间的数）

如果是偶数个数据：
- 数据：`[3, 7, 1, 9, 5, 8]`
- 排序后：`[1, 3, 5, 7, 8, 9]`
- **中值 = (5 + 7) / 2 = 6.0**（中间两个数的平均值）

**中值滤波就是不断用"局部中值"替换原始数据点**，从而达到去噪的目的。

### 1.3 中值滤波的典型应用场景 🎨

中值滤波特别擅长处理**椒盐噪声（Salt-and-Pepper Noise）**，这种噪声表现为图像中随机出现的黑白像素点（像撒了盐和胡椒一样）。

**常见应用场景**：

| 场景 | 说明 | 示例 |
|------|------|------|
| **图像去噪** 🖼️ | 去除传感器故障、传输错误导致的椒盐噪声 | 老照片修复、医学影像处理 |
| **信号处理** 📊 | 滤除一维信号中的脉冲干扰 | 音频降噪、传感器数据平滑 |
| **视频处理** 🎬 | 对视频帧逐帧去噪 | 监控画面增强、视频会议优化 |
| **数据预处理** 🔧 | 清理异常值，提升后续算法效果 | 机器学习特征工程 |

**椒盐噪声示例** 🔍：

假设一张图像的局部像素值（0=黑，255=白）：

```
原始干净图像：          添加椒盐噪声后：
[120, 125, 118]        [120,   0, 118]  ← 125变成了0（黑点，"椒"）
[122, 123, 121]   →    [122, 123, 255]  ← 121变成了255（白点，"盐"）
[119, 124, 120]        [119, 124, 120]
```

中值滤波可以完美修复这些极端值！✨

### 1.4 中值滤波的核心优势 🌟

与传统的线性滤波（如均值滤波、高斯滤波）相比，中值滤波有三大独特优势：

1. **完美去除脉冲噪声** 💥
   - 椒盐噪声是极端值（0或255），中值滤波直接"投票"排除它们
   - 线性滤波会把极端值平均进去，导致噪声扩散

2. **保护边缘细节** 🔪
   - 中值不会模糊图像的锐利边缘
   - 均值滤波会让边缘变模糊（因为做了平均）

3. **非线性特性** 🎭
   - 不依赖固定的数学公式加权，而是基于数据排序
   - 对不同类型的噪声都有鲁棒性

**直观类比** 🗳️：想象班级投票选班长
- **均值滤波**：把所有人的意见"平均"一下 → 极端意见会影响结果
- **中值滤波**：取"最中间"的意见 → 排除极端，反映主流声音

## 2. 核心原理与算法流程 🧮

> 📝 本章深入讲解中值滤波的工作原理、算法步骤和关键细节

### 2.1 算法核心流程 📋

中值滤波的算法流程非常直观，可以拆解为以下步骤：

**步骤1：定义滑动窗口** 🪟
- 选择一个**奇数大小**的窗口（如 3×3、5×5、7×7）
- 窗口中心对准当前要处理的像素点

**步骤2：收集窗口内所有像素值** 📥
- 读取窗口覆盖的所有像素灰度值
- 例如 3×3 窗口会收集 9 个像素值

**步骤3：排序取中值** 📊
- 将这些像素值**从小到大排序**
- 取排序后**正中间**的那个值（中位数）

**步骤4：替换中心像素** 🔄
- 用中值替换窗口中心位置的原始像素值
- 窗口滑动到下一个位置，重复上述步骤

### 2.2 详细示例演示 🌰

让我们用一个具体的 3×3 窗口来演示整个过程：

**原始图像局部（3×3区域）**：

```
[120, 125, 118]
[122,   0, 121]  ← 中心像素是 0（椒盐噪声，异常值）
[119, 124, 120]
```

**步骤1：收集窗口内所有像素值** 📥

```
像素值列表：[120, 125, 118, 122, 0, 121, 119, 124, 120]
```

**步骤2：排序** 📊

```
排序后：[0, 118, 119, 120, 120, 121, 122, 124, 125]
         ↑                              ↑
       最小值                          最大值
```

**步骤3：取中值** 🎯

```
共 9 个值，中值位置 = 9 // 2 = 4（从0开始索引）
中值 = 120（第5个数）
```

**步骤4：替换中心像素** 🔄

```
处理后的图像：
[120, 125, 118]
[122, 120, 121]  ← 0 被替换成了 120（中值）✅
[119, 124, 120]
```

**可以看到**：噪声值 0 被完美修复成了周围像素的合理值 120！✨

### 2.3 窗口大小的选择 📏

窗口大小（kernel size）是中值滤波最关键的参数，直接影响去噪效果和细节保留：

| 窗口大小 | 去噪能力 | 细节保留 | 计算速度 | 适用场景 |
|---------|---------|---------|---------|---------|
| **3×3** | ⭐⭐ | ⭐⭐⭐⭐⭐ | 🚀 最快 | 轻微噪声、实时处理 |
| **5×5** | ⭐⭐⭐⭐ | ⭐⭐⭐ | ⚡ 中等 | 中等噪声、一般场景 |
| **7×7** | ⭐⭐⭐⭐⭐ | ⭐⭐ | 🐢 较慢 | 严重噪声、离线处理 |
| **9×9+** | ⭐⭐⭐⭐⭐⭐ | ⭐ | 🐌 很慢 | 极端噪声、特殊需求 |

**选择原则** 🎯：
- **窗口越大** → 去噪越强，但细节损失越多，计算越慢
- **窗口越小** → 细节保留越好，但去噪能力有限
- **必须为奇数** → 保证有明确的中心像素（如 3, 5, 7）

**直观类比** 🪟：想象透过窗户看风景
- **小窗户（3×3）**：看得清楚细节，但视野窄，可能漏掉远处的噪声
- **大窗户（7×7）**：视野宽广，能排除更多干扰，但细节变模糊

### 2.4 边界处理策略 🖼️

当滑动窗口移动到图像边缘时，窗口会超出图像边界。常见的处理策略有：

| 策略 | 说明 | 示例 | 优缺点 |
|------|------|------|--------|
| **填充0（Zero Padding）** | 边界外填充0 | `[0, 0, 120, ...]` | 简单，但可能引入黑边 |
| **复制边界（Replicate）** | 重复边缘像素值 | `[120, 120, 120, ...]` | 效果好，OpenCV默认 |
| **反射（Reflect）** | 镜像反射边界 | `[..., 122, 120, 122, ...]` | 自然过渡，保持连续性 |
| **忽略边界（Valid）** | 不处理边缘像素 | 输出图像变小 | 最简单，但丢失边界信息 |

**OpenCV 中的实现** 🔧：

```python
import cv2                                                  # 导入OpenCV库 📦

# 使用复制边界策略（默认）
result = cv2.medianBlur(image, ksize=5)                     # ksize=5 表示 5×5 窗口
```

### 2.5 时间复杂度分析 ⏱️

中值滤波的计算复杂度取决于：
- **图像尺寸**：$H \times W$（高度 × 宽度）
- **窗口大小**：$K \times K$（如 3×3、5×5）
- **排序算法**：通常使用快速排序或堆排序

**标准实现的时间复杂度** 📐：

$$
O(H \times W \times K^2 \log(K^2))
$$

**解释**：
- $H \times W$：需要处理每个像素
- $K^2$：窗口内的像素数量
- $\log(K^2)$：排序的时间复杂度

**示例计算** 🧮：
- 图像：$1000 \times 1000$ 像素
- 窗口：$5 \times 5$（25个像素）
- 总操作数：$1000 \times 1000 \times 25 \times \log_2(25) \approx 1000 \times 1000 \times 25 \times 4.64 \approx 116,000,000$ 次操作

**优化方法** 🚀：
- **直方图优化**：利用窗口滑动时的重叠，减少重复排序
- **近似算法**：用快速近似中值代替精确排序
- **并行计算**：利用 GPU 或多线程加速

## 3. 手动实现中值滤波 💻

> 🔧 本章从零编写中值滤波的完整 Python 代码，逐行讲解实现细节

### 3.1 完整代码实现 📝

下面是使用 NumPy 手动实现的中值滤波代码：

```python
import numpy as np                                            # 导入NumPy库，提供高效的数组运算 📊

"""手动实现中值滤波函数 💻

参数:
    image: 输入图像（二维数组）[height, width]
    kernel_size: 窗口大小（必须为奇数，如3, 5, 7）
    
返回:
    filtered_image: 滤波后的图像 [height, width]
    
示例:
    result = manual_median_filter(image, kernel_size=3)
"""
def manual_median_filter(image, kernel_size=3):
    # 1️⃣ 验证窗口大小必须为奇数
    if kernel_size % 2 == 0:                                  # 判断是否为偶数
        raise ValueError("kernel_size 必须为奇数！")            # 抛出异常提示
    
    # 2️⃣ 获取图像尺寸，示例：image.shape=[100, 100] → height=100, width=100
    height, width = image.shape
    
    # 3️⃣ 计算填充大小，示例：kernel_size=3 → pad=1
    pad = kernel_size // 2
    
    # 4️⃣ 对图像进行边界填充（复制边界策略），数据流动：image[100,100] → padded[102,102]
    padded = np.pad(image, pad_width=pad, mode='edge')
    
    # 5️⃣ 创建输出图像（与输入尺寸相同），示例：filtered[100,100]，初始化为0
    filtered = np.zeros_like(image)
    
    # 6️⃣ 遍历每个像素（滑动窗口）
    for i in range(height):                                   # 遍历行，示例：i从0到99
        for j in range(width):                                # 遍历列，示例：j从0到99
            # 7️⃣ 提取当前窗口内的所有像素值
            # 数据流动：padded[i:i+3, j:j+3] → window[3,3] → 9个像素值
            window = padded[i:i+kernel_size, j:j+kernel_size]
            
            # 8️⃣ 将窗口展平为一维数组并排序取中值
            # 数据流动：window[3,3] → flattened[9] → median_value（标量）
            median_value = np.median(window)
            
            # 9️⃣ 将中值赋给输出图像的对应位置
            # 数据流动：filtered[i,j] = median_value
            filtered[i, j] = median_value
    
    return filtered                                           # 返回滤波后的图像
```

### 3.2 代码逐行解析 🔍

> 📖 本节详细拆解每一步的实现逻辑

**第1步：验证窗口大小** 1️⃣

```python
if kernel_size % 2 == 0:                                      # 判断是否为偶数，示例：kernel_size=4 → 4%2=0 → True
    raise ValueError("kernel_size 必须为奇数！")                # 抛出异常，阻止执行
```

**为什么必须是奇数？** 🤔

中值滤波需要一个明确的**中心像素**，只有奇数大小的窗口才有唯一的中心：

| 窗口大小 | 中心位置 | 说明 |
|---------|---------|------|
| **3×3** | 索引 [1,1] | ✅ 明确的中心 |
| **5×5** | 索引 [2,2] | ✅ 明确的中心 |
| **4×4** | 索引 [1.5,1.5] | ❌ 没有整数中心 |

计算中心位置的公式：

$$
\text{center} = \text{kernel\_size} // 2
$$

- kernel_size=3 → center=1（第2个位置）
- kernel_size=5 → center=2（第3个位置）

---

**第2-3步：获取图像尺寸和计算填充** 2️⃣ 3️⃣

```python
height, width = image.shape                                   # 获取图像高度和宽度，示例：image.shape=[100,100]
pad = kernel_size // 2                                        # 计算填充大小，示例：3//2=1, 5//2=2
```

**什么是填充（Padding）？** 🖼️

当窗口滑动到图像边缘时，会超出边界。填充就是在图像周围"补一圈"像素，让窗口能完整覆盖边缘区域。

**示例**：

```
原始图像 [4×4]：        填充后 [6×6]（pad=1）：
[1, 2, 3, 4]           [1, 1, 2, 3, 4, 4]  ← 上下各补1行
[5, 6, 7, 8]    →      [1, 1, 2, 3, 4, 4]
[9, 10, 11, 12]        [5, 5, 6, 7, 8, 8]
[13, 14, 15, 16]       [9, 9, 10, 11, 12, 12]
                       [13, 13, 14, 15, 16, 16]
                       [13, 13, 14, 15, 16, 16]  ← 左右各补1列
```

**第4步：边界填充** 4️⃣

```python
padded = np.pad(image, pad_width=pad, mode='edge')            # 使用复制边界策略填充，数据流动：image[100,100] → padded[102,102]
```

`mode='edge'` 表示**复制边缘像素值**，这是最常用的策略，能避免引入新的噪声。

**其他填充模式对比**：

| mode | 策略 | 效果 | 适用场景 |
|------|------|------|---------|
| `'edge'` | 复制边界 | 自然过渡 | ✅ 最常用 |
| `'constant'` | 填充常数（默认0） | 可能引入黑边 | 特殊需求 |
| `'reflect'` | 镜像反射 | 对称连续 | 周期性信号 |
| `'wrap'` | 循环填充 | 首尾相连 | 纹理图像 |

---

**第5步：创建输出图像** 5️⃣

```python
filtered = np.zeros_like(image)                               # 创建与输入相同尺寸的全零数组，示例：filtered[100,100]
```

`np.zeros_like(image)` 创建一个与 `image` **形状和类型相同**的全零数组，用于存储滤波结果。

**第6-9步：滑动窗口处理** 6️⃣ 7️⃣ 8️⃣ 9️⃣

```python
for i in range(height):                                       # 遍历每一行，示例：i从0到99
    for j in range(width):                                    # 遍历每一列，示例：j从0到99
        window = padded[i:i+kernel_size, j:j+kernel_size]     # 提取窗口，数据流动：padded[102,102] → window[3,3]
        median_value = np.median(window)                      # 计算中值，数据流动：window[3,3] → 9个值 → 标量
        filtered[i, j] = median_value                         # 赋值，数据流动：filtered[i,j] = median_value
```

**窗口提取详解** 🔍：

当 `i=0, j=0, kernel_size=3` 时：

```python
window = padded[0:3, 0:3]                                     # 提取 padded 的左上角 3×3 区域
```

这相当于从填充后的图像中"切"出一个 3×3 的小块：

```
padded 图像：           提取 window：
[1, 1, 2, ...]         [1, 1, 2]
[1, 1, 2, ...]    →    [1, 1, 2]
[5, 5, 6, ...]         [5, 5, 6]
```

**中值计算** 📊：

```python
median_value = np.median(window)
```

`np.median()` 会自动完成以下步骤：
1. 将二维数组展平为一维：`[[1,1,2],[1,1,2],[5,5,6]]` → `[1,1,2,1,1,2,5,5,6]`
2. 排序：`[1,1,1,1,2,2,5,5,6]`
3. 取中间值：中位数 = 2

### 3.3 测试手动实现 🧪

```python
import numpy as np                                            # 导入NumPy库 📊

# 创建一个简单的 5×5 测试图像（带椒盐噪声）
test_image = np.array([
    [120, 125, 118, 122, 119],
    [122,   0, 121, 119, 124],  # 0 是椒盐噪声
    [119, 124, 120, 123, 121],
    [121, 122, 255, 120, 118],  # 255 是椒盐噪声
```

## 5. 中值滤波 vs 均值滤波对比 ⚔️

> 🔍 本章深入对比两种常用滤波器的本质差异、适用场景和性能表现

### 5.1 核心原理对比 📊

**中值滤波** 和 **均值滤波** 是最常用的两种图像平滑技术，但它们的原理和效果截然不同：

| 对比维度 | 中值滤波 🎯 | 均值滤波 📐 |
|---------|-----------|-----------|
| **滤波类型** | 非线性滤波 | 线性滤波 |
| **核心操作** | 排序取中值 | 求算术平均 |
| **数学公式** | $\text{median}(x_1, x_2, \ldots, x_n)$ | $\frac{1}{n}\sum_{i=1}^{n} x_i$ |
| **计算复杂度** | 较高（需要排序） | 较低（只需求和） |
| **边缘保护** | ✅ 优秀 | ❌ 会模糊 |

### 5.2 去噪效果对比 🎨

**场景1：椒盐噪声（脉冲噪声）**

假设 3×3 窗口内的像素值：`[120, 125, 118, 122, 0, 121, 119, 124, 120]`

- **中值滤波**：
  ```
  排序后：[0, 118, 119, 120, 120, 121, 122, 124, 125]
  中值 = 120 ✅ 完美修复噪声
  ```

- **均值滤波**：
  ```
  平均值 = (120+125+118+122+0+121+119+124+120) / 9
         = 969 / 9
         = 107.67 ❌ 被极端值 0 拉低
  ```

**结论** 🎯：中值滤波对椒盐噪声的处理效果远优于均值滤波！

**场景2：高斯噪声（轻微随机噪声）**

假设 3×3 窗口内的像素值：`[120, 123, 119, 121, 122, 120, 118, 124, 121]`

- **中值滤波**：
  ```
  排序后：[118, 119, 120, 120, 121, 121, 122, 123, 124]
  中值 = 121 ✅ 合理
  ```

- **均值滤波**：
  ```
  平均值 = (120+123+119+121+122+120+118+124+121) / 9
         = 1088 / 9
         = 120.89 ✅ 同样合理
  ```

**结论** 📊：对于轻微随机噪声，两者效果相近，但均值滤波计算更快。

### 5.3 边缘保护能力对比 🔪

**边缘保护**是图像处理中的关键指标。让我们通过一个简单示例来看两者的差异：

**示例：图像边缘区域**

```
原始图像（左侧亮，右侧暗）：
[200, 200, 200 | 50, 50, 50]
[200, 200, 200 | 50, 50, 50]  ← 竖线是边缘
[200, 200, 200 | 50, 50, 50]
```

**中值滤波结果（3×3窗口）**：

```
[200, 200, 200 | 50, 50, 50]  ← 边缘保持锐利 ✅
[200, 200, 200 | 50, 50, 50]
[200, 200, 200 | 50, 50, 50]
```

**均值滤波结果（3×3窗口）**：

```
[200, 175, 125 | 100, 75, 50]  ← 边缘变模糊 ❌
[200, 175, 125 | 100, 75, 50]
[200, 175, 125 | 100, 75, 50]
```

**为什么会这样？** 🤔

- **中值滤波**：窗口跨越边缘时，中值要么是 200，要么是 50（取决于哪边像素多），所以边缘保持清晰
- **均值滤波**：窗口跨越边缘时，会把 200 和 50 平均，得到中间值 125，导致边缘模糊

**直观类比** 🎨：想象你在画一条黑线和白线的分界
- **中值滤波**：像用尺子沿着边界画，保持锐利
- **均值滤波**：像用毛刷在边界涂抹，导致模糊

### 5.4 完整对比表格 📋

| 特性 | 中值滤波 | 均值滤波 | 高斯滤波 |
|------|---------|---------|---------|
| **滤波类型** | 非线性 | 线性 | 线性 |
| **椒盐噪声** | ⭐⭐⭐⭐⭐ 极佳 | ⭐⭐ 差 | ⭐⭐ 差 |
| **高斯噪声** | ⭐⭐⭐ 中等 | ⭐⭐⭐⭐ 好 | ⭐⭐⭐⭐⭐ 极佳 |
| **边缘保护** | ⭐⭐⭐⭐⭐ 优秀 | ⭐⭐ 差 | ⭐⭐⭐ 中等 |
| **计算速度** | ⭐⭐ 较慢 | ⭐⭐⭐⭐⭐ 最快 | ⭐⭐⭐⭐ 快 |
| **实现复杂度** | 中等（需排序） | 简单（求和平均） | 中等（权重计算） |
| **适用场景** | 脉冲噪声、边缘敏感 | 轻微噪声、快速处理 | 高斯噪声、平滑需求 |

### 5.5 如何选择？🎯

**选择中值滤波的场景** ✅：
- 图像存在**椒盐噪声**（黑白点）
- 需要**保护边缘细节**（如文字识别、边缘检测预处理）
- 噪声是**脉冲型**的（突然的极端值）

**选择均值滤波的场景** ✅：
- 图像存在**轻微随机噪声**
- 对**计算速度要求高**（实时处理）
- **不关心边缘模糊**（如背景虚化）

**选择高斯滤波的场景** ✅：
- 图像存在**高斯噪声**（传感器热噪声）
- 需要**平滑但保留一定边缘**
- 作为**其他算法的预处理**（如 Canny 边缘检测）

```python

import numpy as np                                          # 导入NumPy库 📊
import matplotlib.pyplot as plt                             # 导入绘图模块 🎨


"""手动实现中值滤波函数 💻

参数:
    image: 输入图像（二维数组）[height, width]
    kernel_size: 窗口大小（必须为奇数，如3, 5, 7）
    
返回:
    filtered_image: 滤波后的图像 [height, width]
    
示例:
    result = manual_median_filter(image, kernel_size=3)
"""
def manual_median_filter(image, kernel_size=3):
    # 1️⃣ 验证窗口大小必须为奇数
    if kernel_size % 2 == 0:                                  # 判断是否为偶数
        raise ValueError("kernel_size 必须为奇数！")            # 抛出异常提示
    
    # 2️⃣ 获取图像尺寸，示例：image.shape=[100,100] → height=100, width=100
    height, width = image.shape
    
    # 3️⃣ 计算填充大小，示例：kernel_size=3 → pad=1
    pad = kernel_size // 2
    
    # 4️⃣ 对图像进行边界填充（复制边界策略），数据流动：image[100,100] → padded[102,102]
    padded = np.pad(image, pad_width=pad, mode='edge')
    
    # 5️⃣ 创建输出图像（与输入尺寸相同），示例：filtered[100,100]，初始化为0
    filtered = np.zeros_like(image)
    
    # 6️⃣ 遍历每个像素（滑动窗口）
    for i in range(height):                                   # 遍历行，示例：i从0到99
        for j in range(width):                                # 遍历列，示例：j从0到99
            # 7️⃣ 提取当前窗口内的所有像素值
            # 数据流动：padded[i:i+3, j:j+3] → window[3,3] → 9个像素值
            window = padded[i:i+kernel_size, j:j+kernel_size]
            
            # 8️⃣ 将窗口展平为一维数组并排序取中值
            # 数据流动：window[3,3] → flattened[9] → median_value（标量）
            median_value = np.median(window)
            
            # 9️⃣ 将中值赋给输出图像的对应位置
            # 数据流动：filtered[i,j] = median_value
            filtered[i, j] = median_value
    
    return filtered                                           # 返回滤波后的图像


"""添加椒盐噪声函数 🎲

参数:
    image: 输入图像 [height, width] 或 [height, width, channels]
    noise_ratio: 噪声比例（0.05表示5%的像素会被污染）
    
返回:
    noisy_image: 带椒盐噪声的图像
    
示例:
    noisy = add_salt_pepper_noise(image, noise_ratio=0.05)
"""
def add_salt_pepper_noise(image, noise_ratio=0.05):
    noisy = image.copy()                                      # 复制图像，保护原始数据 🛡️
    
    # 判断是灰度图还是彩色图
    if len(image.shape) == 2:                                 # 灰度图 [H, W]
        h, w = image.shape                                    # 获取尺寸
        num_noise = int(h * w * noise_ratio)                  # 计算噪声点数量
        xs = np.random.randint(0, w, num_noise)               # 随机x坐标
        ys = np.random.randint(0, h, num_noise)               # 随机y坐标
        for x, y in zip(xs, ys):                              # 遍历噪声点
            noisy[y, x] = 0 if np.random.random() < 0.5 else 255  # 50%概率为0或255
    else:                                                     # 彩色图 [H, W, C]
        h, w, c = image.shape                                 # 获取尺寸和通道数
        num_noise = int(h * w * noise_ratio)                  # 计算噪声点数量
        xs = np.random.randint(0, w, num_noise)               # 随机x坐标
        ys = np.random.randint(0, h, num_noise)               # 随机y坐标
        for x, y in zip(xs, ys):                              # 遍历噪声点
            if np.random.random() < 0.5:                      # 50%概率
                noisy[y, x] = [0, 0, 0]                       # 椒噪声（黑色）
            else:
                noisy[y, x] = [255, 255, 255]                 # 盐噪声（白色）
    
    return noisy                                              # 返回带噪声的图像
```

### 7.1 核心知识点回顾 📚

| 知识点 | 关键内容 | 重要性 |
|--------|---------|--------|
| **基本概念** 🤔 | 非线性滤波、排序取中值、滑动窗口 | ⭐⭐⭐⭐⭐ |
| **算法流程** 🧮 | 窗口滑动 → 收集像素 → 排序 → 取中值 → 替换 | ⭐⭐⭐⭐⭐ |
| **窗口大小** 📏 | 必须为奇数，3×3/5×5/7×7，越大去噪越强但越模糊 | ⭐⭐⭐⭐ |
| **边界处理** 🖼️ | 复制边界、填充0、反射、忽略边界 | ⭐⭐⭐ |
| **手动实现** 💻 | NumPy实现理解原理，Python循环性能较低 | ⭐⭐⭐⭐ |

### 7.2 关键理解 💡

**中值滤波的本质** 🎯：
- 不是“平均”，而是“投票”——选最中间的值
- 极端值（噪声）会被排到两端，不影响中值结果
- 这是一种**基于排序的非线性操作**，不是数学加权平均

**为什么中值滤波能保护边缘？** 🔪：
- 窗口跨越边缘时，中值要么是左侧值，要么是右侧值
- 不会像均值那样“混合”两侧的值
- 因此边缘保持锐利，不会模糊

**什么时候用中值滤波？** ✅：
- 图像有**椒盐噪声**（黑白点）
- 需要**保留边缘细节**（文字、线条、轮廓）
- 噪声是**脉冲型**的（突然的极端值）

**什么时候不用中值滤波？** ❌：
- 图像有**高斯噪声**（轻微随机噪声）→ 用高斯滤波
- 对**计算速度要求极高**（实时视频）→ 用均值滤波
- 需要**创造新像素值**（插值、放大）→ 中值不创造新值

### 7.3 最佳实践建议 🌟

| 场景 | 推荐方案 | 窗口大小 | 说明 |
|------|---------|---------|------|
| **轻微椒盐噪声** | 中值滤波 | 3×3 | 快速去噪，保留细节 |
| **中等椒盐噪声** | 中值滤波 | 5×5 | 平衡去噪和细节 |
| **严重椒盐噪声** | 中值滤波 | 7×7 或更大 | 强力去噪，可能损失细节 |
| **彩色图像** | OpenCV中值滤波 | 5×5 | 自动处理多通道 |
| **实时处理** | OpenCV中值滤波 | 3×3 | 性能最优 |
| **学习理解** | 手动实现 | 3×3 | 理解每步原理 |

### 7.4 常见问题与陷阱 ⚠️

**Q1：窗口大小必须是奇数吗？** 🤔

**A**：是的！必须为奇数（3, 5, 7...），因为只有奇数窗口才有明确的中心像素。偶数窗口会导致中心位置不明确。

**Q2：中值滤波能去除所有类型的噪声吗？** 🤔

**A**：不是。中值滤波最擅长去除**椒盐噪声**（脉冲噪声），对高斯噪声（轻微随机噪声）效果一般，对周期性噪声几乎无效。

**Q3：为什么OpenCV比手动实现快这么多？** 🤔

**A**：三个原因：
1. **C++底层**：避免Python解释器开销
2. **SIMD指令**：CPU向量并行处理
3. **算法优化**：直方图优化、减少重复排序

**Q4：彩色图像怎么处理？** 🤔

**A**：OpenCV的`cv2.medianBlur()`会**自动对每个通道单独处理**，然后堆叠回彩色图像。不需要手动拆分通道。

**Q5：窗口越大越好吗？** 🤔

**A**：不是！窗口越大：
- ✅ 去噪能力越强
- ❌ 细节损失越多
- ❌ 计算越慢
- 需要根据噪声程度和细节需求**权衡选择**
