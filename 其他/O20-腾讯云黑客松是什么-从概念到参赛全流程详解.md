## 1. 什么是黑客松 🧩 / What Is a Hackathon

> 📦 **Note:** 本章介绍黑客松的概念起源与核心特点 / This chapter introduces the concept, origin, and core features of hackathons.

**黑客松**（Hackathon），又称"编程马拉松"，是由 **Hack**（编程创造）和 **Marathon**（马拉松）两个词组合而成的合成词，指在 **限定时间内** ，聚集一批开发者，以紧密协作的方式完成从创意构思到作品开发的全流程活动。

**直观类比** 🎯：想象一场"创意冲刺跑"——就像马拉松选手要在规定时间内跑完全程，黑客松参赛者要在规定时间内把一个想法变成一个可以运行的作品。

### 1.1 起源与发展 📜 / Origin and Development

黑客松最早可追溯到 **1999 年** ，由两个团队分别提出：

- **OpenBSD 团队**：1999 年 6 月 4 日在卡尔加里举办密码学开发活动时首次使用该术语
- **Sun（昇阳电脑）**：1999 年 6 月 19 日在 JavaOne 大会中发起了一场用 Java 为 Palm V 编写程序的编程挑战

此后，黑客松在全球科技行业迅速流行。**Facebook** 是将黑客松文化发扬光大的代表——其著名的"点赞（Like）"功能、第一款视频播放器、聊天系统等重要产品，都诞生于公司内部的黑客松活动。

**直观类比** 🏠：黑客松就像一个"创新孵化器"——平时需要几个月才能完成的创意原型，在高强度的集中开发下，几十个小时就能变成现实。

### 1.2 核心特点 ⚡ / Core Features

黑客松区别于普通编程比赛的核心特征：

| 特点 | 说明 |
|------|------|
| **限时开发** ⏰ | 通常在 24-120 小时内完成，从概念到作品 |
| **团队协作** 👥 | 2-5 人组队，成员专业互补（程序员、设计师、策划等） |
| **从零到一** 🚀 | 从零开始开发，当场提交可运行的作品 |
| **主题驱动** 🎯 | 围绕特定主题或赛道进行创作 |
| **现场演示** 🎤 | 最终需要路演展示，接受评委评审 |

**谁能参加？** 🤔

黑客松并非只有程序员才能参加。一个完整的团队通常需要：

- **程序员** 💻：负责编码开发
- **设计师** 🎨：负责 UI/UX 和视觉设计
- **创意策划** 💡：负责产品概念和创意方向
- **领域专家** 🔬：提供特定领域的专业知识

只要对技术有热情、能提出创意想法，任何人都可以成为黑客松的一份子。

## 2. 腾讯云黑客松概述 🏆 / Overview of Tencent Cloud Hackathon

> 📦 **Note:** 本章介绍腾讯云黑客松的赛事背景与核心定位 / This chapter introduces the background and core positioning of Tencent Cloud Hackathon.

**腾讯云黑客松**（Tencent Cloud Hackathon）是由 **腾讯云** 主办的全球性 AI 编程赛事品牌，官网地址为 [tch.cloud.tencent.com](https://tch.cloud.tencent.com/)。

赛事以 **AI 驱动创新** 为核心定位，联合腾讯游戏学堂、腾讯 WeTech Academy、腾讯互娱社会价值探索中心、腾讯教育等多个部门共同举办，面向全球高校开发者、企业开发者及个人开发者开放。

### 2.1 赛事体系 🏗️ / Competition System

腾讯云黑客松并非单一赛事，而是一个包含多种赛事形式的体系：

| 赛事类型 | 说明 |
|---------|------|
| **线上公开挑战赛** 🌐 | 线上提交作品，面向全球开发者 |
| **线下极限黑客松** 🏟️ | 在腾讯滨海大厦等地举办，限时现场开发 |
| **全球路演** 🌍 | 分赛区路演 + 总决赛，现场展示与评审 |

### 2.2 近期主要赛事 📅 / Recent Major Events

截至 2026 年，腾讯云黑客松正在举办或已完成的主要赛事包括：

**（1）游戏开发挑战赛（AI CAN DO IT）** 🎮

- **奖金**：100 万元
- **时间**：2026.04.17 - 2026.09.01
- **主题**：AI 驱动的游戏创作
- **面向**：全球高校开发者
- **主办方**：腾讯云、腾讯游戏学堂、Tencent WeTech Academy、腾讯互娱社会价值探索中心、腾讯教育

**（2）AI 智能体争霸赛** 🤖

- **奖金**：45 万元
- **时间**：2026.04.15 - 2026.09.18
- **主题**：AI 智能体（Agent）应用开发
- **核心**：以赛代训，验证 CodeBuddy、ClawPro、WorkBuddy 等工具能力
- **赛道**：双赛道（Skill + Agent）、五大赛区路演

**（3）AI Coding Hackathon Singapore 2026** 🇸🇬

- **奖金**：USD $1,700 credit
- **时间**：2026.03.26 - 2026.05.09
- **主题**：构建能思考、决策和执行的 AI Agent
- **面向**：开发者、创始人、学生

## 3. 赛道设置 🛤️ / Track Settings

> 📦 **Note:** 本章介绍腾讯云黑客松的主要赛道方向与创作要求 / This chapter introduces the main track directions and creative requirements.

以 2026 年游戏开发挑战赛为例，赛事设置了 **三大赛题方向** ，参赛者任选其一进行创作：

### 3.1 三大赛题方向 🎯 / Three Track Directions

**（1）小红花游戏 🌸** —— 用游戏传递公益力量

围绕应急急救、碳中和、生物多样性、适老化关怀等主题，让玩家在游戏中感受社会温度。

**直观类比** ❤️：就像用游戏讲一个温暖的故事，让玩家在娱乐中关注社会议题。

**（2）文化表达游戏 🏮** —— 用游戏讲述文化故事

聚焦传统文化焕新、城市与乡村文化、民族地域特色等，让文化在游戏中活起来。

**（3）叙事剧情游戏 📖** —— 用 AI 重塑叙事体验

以经营模拟、剧情驱动、开放探索等形式，用智能 NPC 和 AI 世界观打造全新游戏叙事体验。

### 3.2 AI 创作要求 🤖 / AI Creation Requirements

作品需在以下 **至少一个环节** 实现 AI 创作：

| 环节 | 说明 |
|------|------|
| **世界观 & 剧情** 📝 | 由 AI 生成游戏世界观、背景故事、历史脉络、势力关系与规则体系 |
| **游戏原画** 🎨 | 由 AI 生成游戏角色设计、场景概念图、UI 美术等主要视觉资产 |
| **游戏安全体系** 🔒 | 由 AI 生成游戏 IOA 安全体系，覆盖 Agent 行为安全、玩家身份鉴权等 |
| **声音表演** 🎵 | 由 AI 生成游戏音乐、音效与配音，构建沉浸式听觉叙事 |

### 3.3 AI 智能体争霸赛赛道 🤖 / AI Agent Track

AI 智能体争霸赛则设置了不同的赛道方向：

| 赛道 | 说明 |
|------|------|
| **企业微信专属智能体赛道** 💼 | 聚焦企业级场景，开发提升办公协作效率的智能解决方案 |
| **微信公众号智能体赛道** 📱 | 面向内容服务领域，探索智能交互在内容创作、用户运营中的创新应用 |
| **创新创意探索赛道** 💡 | 不限行业方向，鼓励开发者突破边界，打造前瞻性 AI 原生应用 |

## 4. 参赛流程 📋 / Participation Process

> 📦 **Note:** 本章介绍从报名到决赛的完整步骤 / This chapter introduces the complete steps from registration to the final.

### 4.1 报名阶段 📝 / Registration Phase

**团队要求** 👥：

- 团队人数 **1-3 人** ，不限专业与年级
- 每队可配 1 名指导老师（高校赛）
- 团队成员专业背景互补效果更佳（程序员 + 设计师 + 策划）

**报名方式** 🔗：

- 通过大赛官网 [tch.cloud.tencent.com](https://tch.cloud.tencent.com/) 完成报名
- 或通过大赛指定二维码完成团队报名
- 报名后需填写团队信息、成员基本信息、腾讯云 ID 等

### 4.2 资源领取 🎁 / Resource Collection

报名成功后，参赛者将获得：

- **CodeBuddy Credits**：国内赛区每人 3000 Credits，港澳及东南亚赛区每人 2000 Credits
- **创作资源包**：包含开发工具、SDK、文档等
- **技术支持**：可加入官方赛事群获取帮助

### 4.3 创作开发 💻 / Development Phase

这是黑客松的核心阶段：

1. **创意构思** 💡：确定作品主题、核心玩法、技术方案
2. **原型搭建** 🏗️：快速搭建最小可运行版本
3. **功能开发** ⚙️：使用 CodeBuddy 等 AI 工具完成完整开发
4. **测试优化** 🔧：调试运行、优化体验
5. **材料准备** 📦：录制 Demo 视频、制作 PPT、部署在线链接

**直观类比** 🍳：就像做菜——先想好菜单（创意），准备食材（资源），然后洗切炒煮（开发），最后摆盘上桌（提交作品）。

### 4.4 线上训练营 📚 / Online Training Camp

贯穿开发期的"学 + 诊"双轨机制，共 8 期，每周 1 期：

- 赛题精读
- AI 原型快速搭建
- AI 生成游戏世界观设定
- 每期开放 10 个作品诊断名额，可获得嘉宾针对性点评

### 4.5 赛区路演 🎤 / Regional Roadshow

各赛区前十名晋级线下路演：

| 赛区 | 路演时间 | 举办地点 |
|------|---------|---------|
| 华北赛区 | 6 月 | 北京 |
| 港澳赛区 | 7 月 | 香港 |
| 华东赛区 | 7 月 | 上海 |
| 华南赛区 | 7 月 | 武汉 |
| 东南亚赛区 | 7 月 | 印度尼西亚 |
| **总决赛** | **9 月** | **深圳（腾讯全球数字生态大会）** |

各赛区前三名晋级深圳总决赛，参与限时开发及全球路演。

## 5. 作品提交要求 📦 / Submission Requirements

> 📦 **Note:** 本章介绍参赛队伍需要提交的具体材料清单 / This chapter introduces the specific materials that participating teams need to submit.

每个参赛队伍需提交以下材料：

| 材料 | 说明 |
|------|------|
| **作品网页链接** 🔗 | 部署至浏览器环境的可访问在线链接（如 Cloud Studio 等） |
| **作品标题** 📌 | 游戏项目或应用名称 |
| **游戏 Demo 视频** 🎬 | 展示核心玩法与 AI 功能的演示视频 |
| **作品介绍 PPT** 📊 | 包含作品简介、AI 创作说明、团队信息等 |
| **CodeBuddy 历史对话** 💬 | 开发过程中的 AI 对话记录 |
| **社交媒体发布链接** 📱 | 加分项：在小红书 / YouTube / X / 视频号发布作品，携带话题标签 |

**关键注意事项** ⚠️：

- 作品必须是 **可运行的完整 Demo** ，不能只是概念或设计稿
- 作品需在 **至少一个环节** 使用了 AI 创作
- 在线链接必须在评审期间 **持续可访问**
- 所有材料以 **在线文档**（推荐腾讯文档）方式提交
