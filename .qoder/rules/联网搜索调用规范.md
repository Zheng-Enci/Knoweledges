## 1. 核心工具概览 📚

> 本章介绍联网搜索涉及的两个核心工具及其定位

联网搜索主要依赖两个工具配合使用：

| 工具 | 功能 | 适用场景 |
|------|------|---------|
| `search_web` | 搜索引擎查询，返回多个网页摘要和URL | 需要查找多个来源、对比信息、获取最新资料 |
| `fetch_content` | 提取指定URL的网页内容 | 已知道具体URL，需要深入阅读某个页面 |

**工具选择原则**：
- **先用 search_web**：获取多个候选来源
- **后用 fetch_content**：对重要链接深入提取内容

## 2. search_web工具调用规范 🔍

> 本章详细说明search_web工具的标准调用格式和参数

### 2.1 基本调用格式

```python
search_web(
  query="搜索关键词",           # 必填：搜索查询词
  timeRange="时间范围"          # 可选：OneDay/OneWeek/OneMonth/OneYear/NoLimit
)
```

### 2.2 参数说明

**query（必填）**
- **类型**：字符串
- **说明**：搜索查询词，应包含核心关键词
- **要求**：
  - 使用简洁明确的技术术语
  - 包含关键实体（如技术名称、版本等）
  - 避免口语化表达

**timeRange（可选）**
- **类型**：枚举字符串
- **说明**：限制搜索结果的时间范围
- **可选值**：
  - `OneDay`：过去24小时
  - `OneWeek`：过去一周
  - `OneMonth`：过去一个月
  - `OneYear`：过去一年
  - `NoLimit`：不限制时间（默认）

### 2.3 调用示例

**示例1：搜索最新技术实践**
```python
search_web(
  query="Python async await 最佳实践 2024",
  timeRange="OneYear"
)
```

**示例2：搜索紧急漏洞信息**
```python
search_web(
  query="Log4j漏洞 CVE-2021-44228 修复方案",
  timeRange="OneWeek"
)
```

**示例3：搜索经典理论（不限时间）**
```python
search_web(
  query="Transformer注意力机制原理",
  timeRange="NoLimit"
)
```

### 2.4 query编写规范

**✅ 正确示例**：
- `Python asyncio 协程 并发编程 最佳实践`
- `Spring Boot 3.0 迁移指南 JDK17`
- `PyTorch 2.0 scaled_dot_product_attention FlashAttention`

**❌ 错误示例**：
- `python怎么异步`（太口语化）
- `springboot新版本怎么用`（不够具体）
- `pytorch attention`（缺少关键细节）

**编写原则**：
1. **技术术语优先**：使用官方文档中的标准术语
2. **版本信息明确**：涉及版本时明确标注（如 2024、3.0、JDK17）
3. **关键词组合**：3-5个核心关键词，用空格分隔
4. **避免疑问句**：使用陈述式关键词而非问题形式

## 3. fetch_content工具调用规范 📄

> 本章详细说明fetch_content工具的标准调用格式和参数

### 3.1 基本调用格式

```python
fetch_content(
  url="完整URL地址",            # 必填：要提取内容的网页URL
  query="内容关键词"            # 可选：希望在页面中查找的内容
)
```

### 3.2 参数说明

**url（必填）**
- **类型**：字符串
- **说明**：完整的网页URL地址
- **要求**：
  - 必须是有效的HTTP/HTTPS链接
  - 包含完整的协议前缀（http://或https://）
  - 不能是相对路径

**query（可选）**
- **类型**：字符串
- **说明**：内容过滤关键词，帮助提取相关部分
- **要求**：
  - 使用与目标内容相关的关键词
  - 可以帮助聚焦长文档的特定章节

### 3.3 调用示例

**示例1：提取官方文档**
```python
fetch_content(
  url="https://docs.python.org/3/library/asyncio.html",
  query="async await coroutine"
)
```

**示例2：提取技术博客**
```python
fetch_content(
  url="https://jalammar.github.io/illustrated-transformer/",
  query="attention mechanism self-attention"
)
```

**示例3：完整URL（无过滤）**
```python
fetch_content(
  url="https://pytorch.org/docs/stable/generated/torch.nn.functional.scaled_dot_product_attention.html"
)
```

## 4. 搜索策略与工作流程 🔄

> 本章介绍从搜索到信息整合的完整工作流程

### 4.1 标准搜索流程

```
1. search_web（获取多个候选来源）
   ↓
2. 筛选高质量结果（官方文档、权威博客、最新内容）
   ↓
3. fetch_content（对重要链接深入提取）
   ↓
4. 综合多个来源信息
   ↓
5. 用自己的话重新表述
   ↓
6. 按规范添加参考资料链接
```

### 4.2 搜索策略选择

**场景1：学习新技术**
```python
# 第1步：搜索概述
search_web(query="FastAPI 入门教程 2024", timeRange="OneYear")

# 第2步：搜索官方文档
search_web(query="FastAPI official documentation getting started", timeRange="NoLimit")

# 第3步：对重要链接深入阅读
fetch_content(url="https://fastapi.tiangolo.com/tutorial/", query="first steps")
```

**场景2：解决具体问题**
```python
# 第1步：搜索错误信息
search_web(query="PyTorch CUDA out of memory 解决方案", timeRange="OneYear")

# 第2步：搜索最佳实践
search_web(query="PyTorch 显存优化 技巧", timeRange="OneYear")

# 第3步：提取详细解决方案
fetch_content(url="https://...", query="memory optimization tips")
```

**场景3：验证事实**
```python
# 直接搜索验证
search_web(query="Python 3.12 新特性", timeRange="OneYear")

# 提取官方发布说明
fetch_content(url="https://docs.python.org/3/whatsnew/3.12.html")
```

### 4.3 搜索结果筛选标准

**高质量来源特征**：
1. **官方文档**：python.org、pytorch.org、spring.io等
2. **权威技术博客**：知名开发者、公司技术博客
3. **最新内容**：时间近、适配当前版本
4. **内容详实**：有示例、有深度、非泛泛而谈
5. **可验证**：代码可运行、有据可查

**低质量来源特征**：
1. 内容过于简短或泛泛而谈
2. 明显过时（如旧版本API）
3. 代码示例无法运行
4. 存在明显技术错误
5. 广告过多或SEO优化痕迹明显

## 5. 时间范围选择指南 ⏰

> 本章指导如何根据搜索内容选择合适的时间范围

### 5.1 时间范围对照表

| 时间范围 | 适用场景 | 示例 |
|---------|---------|------|
| `OneDay` | 紧急漏洞、突发新闻、当天发布 | 安全漏洞、重大事故 |
| `OneWeek` | 最新版本发布、短期问题 | 本周发布的版本 |
| `OneMonth` | 近期最佳实践、趋势分析 | 本月技术讨论 |
| `OneYear` | 技术教程、版本迁移、当前实践 | 2024年的最佳实践 |
| `NoLimit` | 经典理论、基础概念、历史资料 | 算法原理、设计模式 |

### 5.2 选择决策树

```
需要搜索的内容是什么？
├─ 安全漏洞/紧急问题？ → OneDay 或 OneWeek
├─ 最新版本相关？ → OneMonth 或 OneYear
├─ 教程/最佳实践？ → OneYear
└─ 经典理论/基础知识？ → NoLimit
```

### 5.3 实际案例

**案例1：Python新版本特性**
```python
# Python 3.13刚发布，搜索最新特性
search_web(query="Python 3.13 新特性", timeRange="OneMonth")
```

**案例2：框架迁移指南**
```python
# Spring Boot 3.0迁移，需要较新的资料
search_web(query="Spring Boot 2.x 迁移 3.0 指南", timeRange="OneYear")
```

**案例3：经典算法原理**
```python
# Transformer注意力机制是经典理论，不限时间
search_web(query="Transformer self-attention mechanism原理", timeRange="NoLimit")
```

## 6. 搜索结果处理规范 📝

> 本章规范搜索结果的使用、验证和引用方式

### 6.1 信息验证原则

**必须验证的内容**：
1. **代码示例**：确保语法正确、可运行
2. **版本号**：确认API在指定版本存在
3. **配置参数**：验证参数名称和取值范围
4. **技术事实**：交叉验证多个来源

**验证方法**：
- 对比官方文档
- 查看多个独立来源
- 检查评论区和反馈
- 注意内容的发布日期

### 6.2 信息整合规范

**✅ 正确做法**：
1. 综合多个来源的信息
2. 用自己的话重新表述
3. 保留核心知识点
4. 添加实际应用示例
5. 标注所有参考资料

**❌ 错误做法**：
1. 直接复制粘贴原文
2. 只参考单一来源
3. 不验证信息准确性
4. 忽略发布时间（使用过时内容）
5. 不标注引用来源

### 6.3 引用格式规范

搜索结果必须在引用处添加参考资料链接，格式如下：

```markdown
内容段落（联网搜索后编写的内容）...

参考资料：
- [标题 -- 平台名](URL) ⭐值得阅读
- [标题 -- 平台名](URL)
```

**详细格式要求**请参考 [参考链接规范](./参考链接规范.md)。

### 6.4 搜索记录规范

对于复杂的技术调研，建议记录搜索过程：

```markdown
搜索记录：
1. search_web("Python async await 最佳实践 2024", OneYear)
   - 找到官方文档：https://docs.python.org/3/library/asyncio.html
   - 找到技术博客：https://realpython.com/async-io-python/

2. fetch_content("https://docs.python.org/3/library/asyncio.html", "async await")
   - 提取了asyncio核心概念和示例

3. 综合整理，编写文档...
```

## 7. 常见错误与注意事项 ⚠️

> 本章列出联网搜索时的常见错误和注意事项

### 7.1 常见错误

**错误1：query过于宽泛**
```python
# ❌ 错误：太宽泛
search_web(query="Python", timeRange="NoLimit")

# ✅ 正确：具体明确
search_web(query="Python asyncio 异步编程 入门教程", timeRange="OneYear")
```

**错误2：忽略时间范围**
```python
# ❌ 错误：搜索最新技术但不限时间
search_web(query="FastAPI 2024 新特性")  # 默认NoLimit可能找到旧内容

# ✅ 正确：明确时间范围
search_web(query="FastAPI 2024 新特性", timeRange="OneYear")
```

**错误3：不验证直接引用**
```python
# ❌ 错误：搜索到内容后直接复制
search_web(...)
# 直接复制搜索结果到文档

# ✅ 正确：验证后重新表述
search_web(...)
fetch_content(...)  # 深入阅读
# 验证信息准确性
# 用自己的话重新编写
# 添加引用链接
```

### 7.2 注意事项

1. **网络依赖**：联网搜索需要网络连接，离线环境无法使用
2. **结果不确定性**：搜索结果可能因时间、地域等因素不同
3. **内容时效性**：技术内容更新快，注意检查发布时间
4. **版权意识**：不可直接复制他人内容，必须重新表述
5. **引用规范**：所有引用必须按格式添加链接
6. **质量优先**：宁可少而精，不要多而泛

### 7.3 搜索失败处理

如果搜索结果为空或质量不高：

1. **调整query**：更换关键词、使用同义词
2. **扩大时间范围**：从OneYear改为NoLimit
3. **使用英文**：尝试英文关键词（技术内容英文资源更丰富）
4. **分段搜索**：将复杂问题拆分为多个简单问题
5. **查阅官方文档**：直接访问官方文档网站
