# 联网搜索调用规范 🔍

## 前置要求

**每次调用联网搜索前，必须使用命令行获取当前时间**：

⚠️ **重要**：禁止使用内置时间函数或假设当前时间，必须通过执行命令行命令获取实时时间。

```bash
# Windows PowerShell
Get-Date -Format "yyyy年MM月dd日"

# Linux/Mac
date +"%Y年%m月%d日"
```

获取当前时间后，根据年份选择合适的搜索关键词和时间范围。

---

## 核心原则

搜索时应传入**具体的技术术语和关键词组合**，而非口语化表达或疑问句。

---

## Query 编写规范

### 示例说明

如果要搜索某个知识点，应该传入**具体的技术术语和关键词组合**。

**例如**：要搜索"Python函数"这个知识点

❌ **错误**：
```python
query="python函数怎么用"
query="什么是python函数"
query="python函数"
```

✅ **正确**：
```python
query="Python 函数定义 参数传递 返回值 作用域"
query="Python 函数 def 关键字 位置参数 关键字参数"
query="Python 函数 局部变量 全局变量 LEGB规则"
```

**核心原则**：
- 使用技术术语，不用口语化表达
- 3-5个核心关键词，用空格分隔
- 包含该知识点的关键概念和子主题
- 避免疑问句，使用陈述式关键词

### ❌ 错误类型

```python
# 太口语化
query="python怎么异步"
query="springboot新版本怎么用"

# 缺少关键细节
query="pytorch attention"
query="docker漏洞"

# 疑问句形式
query="什么是TCP三次握手"
query="如何优化MySQL性能"
```

## 时间范围选择

根据搜索内容类型选择 `timeRange`：

| 场景 | 推荐值 | 示例 |
|------|--------|------|
| 最新bug/安全漏洞 | `OneDay` 或 `OneWeek` | `"CVE-2024 漏洞修复"` |
| 快速迭代的技术 | `OneMonth` | `"React 19 新特性"` |
| 年度总结/版本更新 | `OneYear` | `"Python 3.12 特性 2024"` |
| 经典知识/基础概念 | `NoLimit` | `"TCP三次握手原理"` |
| 技术演进/发展历史 | `NoLimit` | `"Transformer发展历史"` |

## 搜索策略

### 标准流程

```
1. search_web 获取多个来源摘要
   ↓
2. 筛选高质量结果（官方文档、权威博客、最新内容）
   ↓
3. fetch_content 深入获取重要链接的详细内容
   ↓
4. 综合多个来源，用自己的话重新表述
   ↓
5. 按参考链接规范添加参考资料
```

### 高质量结果判断

**核心标准**（按优先级）：
1. **内容准确性** - 详实清晰、代码可运行、无技术错误
2. **实践价值** - 有可操作示例、真实场景案例
3. **深度与完整性** - 系统讲解、涵盖关键细节和陷阱
4. **时效性** - 技术内容优先最新的
5. **来源权威性** - 官方文档、知名平台（辅助参考）
6. **表达易懂性** - 通俗易懂、有类比图解（加分项）

---

## 注意事项

1. **搜索词要具体**：避免宽泛词汇
2. **时间范围要合理**：不盲目使用最新时间
3. **多渠道验证**：重要信息交叉验证
4. **内容复述**：不直接复制，用自己的话表述
5. **链接格式统一**：严格按参考链接规范
6. **质量优先**：宁缺毋滥

---

## 搜索数量要求

⚠️ **重要**：每次搜索必须获取足够多的相关资料，**越多越好**。

**具体要求**：
- 不要满足于前3-5个搜索结果
- 应该浏览和提取**至少10个以上**的相关来源
- 对于重要技术主题，建议收集**15-20个**或更多资料
- 从多个平台获取信息（官方文档、技术博客、社区讨论、视频教程等）
- 确保覆盖不同的视角和解释方式

**原因**：
- 更多的资料 = 更全面的信息覆盖
- 可以交叉验证准确性
- 能够找到最佳实践和最权威的来源
- 避免单一来源的偏见或错误
