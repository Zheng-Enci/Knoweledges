# 联网搜索调用规范 🔍

## 工具调用说明

联网搜索使用 `search_web` 工具，用于获取最新的网络信息和参考资料。

---

## 调用格式

```python
search_web(
  query="搜索关键词",           # 必需：搜索查询词
  timeRange="时间范围"          # 可选：OneDay/OneWeek/OneMonth/OneYear/NoLimit
)
```

---

## 参数说明

### 1. query（必需）

**类型**：字符串

**说明**：搜索查询词，1到100个字符

**要求**：
- 使用简洁明确的关键词组合
- 包含核心技术术语
- 可以包含年份以获取最新信息

**示例**：
```python
query="Python async await 最佳实践 2024"
query="Transformer attention mechanism optimization"
query="Spring Boot 3.0 new features"
```

### 2. timeRange（可选）

**类型**：枚举字符串

**可选值**：
- `OneDay` - 最近24小时的内容
- `OneWeek` - 最近一周的内容
- `OneMonth` - 最近一个月的内容
- `OneYear` - 最近一年的内容
- `NoLimit` - 不限制时间范围（默认值）

**选择策略**：

| 场景 | 推荐值 | 示例 |
|------|--------|------|
| 最新bug/安全漏洞 | `OneDay` 或 `OneWeek` | "CVE-2024 漏洞修复" |
| 快速迭代的技术 | `OneMonth` | "React 19 新特性" |
| 年度总结/版本更新 | `OneYear` | "Python 3.12 特性 2024" |
| 经典知识/基础概念 | `NoLimit` | "什么是TCP三次握手" |
| 技术演进/发展历史 | `NoLimit` | "Transformer发展历史" |

## 搜索策略

### 1. 搜索流程

```
1. 使用 search_web 获取多个来源的摘要
   ↓
2. 筛选高质量结果（官方文档、权威博客、最新内容）
   ↓
3. 对重要链接使用 fetch_content 深入获取详细内容
   ↓
4. 综合多个来源，用自己的话重新表述
   ↓
5. 按参考链接规范添加参考资料链接
```

### 2. 高质量结果判断标准

**核心标准**（按优先级排序）：

1. **内容准确性**
   - 内容详实、表达清晰，不是泛泛而谈
   - 有据可查的观点，代码示例可运行
   - 逻辑严密，没有明显的技术错误

2. **实践价值**
   - 能解决实际问题，有可操作的示例
   - 提供了真实场景的应用案例
   - 代码和配置经过验证，可以直接使用

3. **深度与完整性**
   - 系统性地讲解知识点，不是碎片化信息
   - 涵盖了关键细节和常见陷阱
   - 提供了原理分析和最佳实践

4. **时效性**
   - 对于技术类内容，优先选择最新的
   - 注意区分内容是否过时（如旧版本API、已废弃的方案）

5. **来源权威性**（辅助参考）
   - 官方文档、知名学术平台可作为加分项
   - 但不应以此排斥小众平台的优质内容
   - 重点仍然看内容本身的质量

6. **表达易懂性**（加分项）
   - 能把复杂的概念用通俗易懂的语言解释清楚
   - 使用生动的类比、图解或示例帮助理解
   - 对难点和易错点有专门的解释和提醒

### 3. 高质量链接标注

对于符合上述高质量标准的参考资料，在链接旁边添加 `⭐值得阅读` 标注：

```markdown
- [The Illustrated Transformer -- Jay Alammar](https://jalammar.github.io/illustrated-transformer/) ⭐值得阅读
```

## 后续操作

### 获取详细内容

使用 `fetch_content` 工具获取特定网页的详细内容：

```python
fetch_content(
  url="https://docs.python.org/3/library/asyncio.html",
  query="async await coroutine"  # 想在页面中找的内容
)
```

### 添加参考资料

按照参考链接规范，在文档中添加引用：

```markdown
参考资料：
- [标题 -- 平台名](URL) ⭐值得阅读
- [标题 -- 平台名](URL)
```

**格式要求**：
- 标题和平台名之间必须使用 ` -- `（两边各一个空格）
- 紧跟引用内容后面（内容写完后下一行就是参考资料）
- 不要放在章节末尾

---

## 注意事项

1. **搜索词要具体**：避免过于宽泛的搜索词，如"Python教程"，应该用"Python async await 协程最佳实践"

2. **时间范围要合理**：根据内容类型选择合适的时间范围，不要盲目使用最新时间

3. **多渠道验证**：重要的技术信息应该从多个来源交叉验证

4. **内容复述**：不可以直接复制他人的语句，要用自己的话重新表述内容

5. **链接格式统一**：严格按照参考链接规范格式化所有引用链接

6. **质量优先**：宁缺毋滥，只添加高质量的参考资料
