# 联网搜索与参考链接规范 🔍

## 前置要求

**每次调用联网搜索前，必须使用命令行获取当前时间**：

⚠️ **重要**：禁止使用内置时间函数或假设当前时间，必须通过执行命令行命令获取实时时间。

```bash
# Windows PowerShell
Get-Date -Format "yyyy年MM月dd日"

# Linux/Mac
date +"%Y年%m月%d日"
```

获取当前时间后，根据年份选择合适的搜索关键词和时间范围。

---

## 第一部分：联网搜索规范

### 核心原则

搜索时应传入**具体的技术术语和关键词组合**，而非口语化表达或疑问句。

---

### Query 编写规范

#### 示例说明

如果要搜索某个知识点，应该传入**具体的技术术语和关键词组合**。

**例如**：要搜索"Python函数"这个知识点

❌ **错误**：
```python
query="python函数怎么用"
query="什么是python函数"
query="python函数"
```

✅ **正确**：
```python
query="Python 函数定义 参数传递 返回值 作用域"
query="Python 函数 def 关键字 位置参数 关键字参数"
query="Python 函数 局部变量 全局变量 LEGB规则"
```

**核心原则**：
- 使用技术术语，不用口语化表达
- 3-5个核心关键词，用空格分隔
- 包含该知识点的关键概念和子主题
- 避免疑问句，使用陈述式关键词

#### ❌ 错误类型

```python
# 太口语化
query="python怎么异步"
query="springboot新版本怎么用"

# 缺少关键细节
query="pytorch attention"
query="docker漏洞"

# 疑问句形式
query="什么是TCP三次握手"
query="如何优化MySQL性能"
```

---

### 时间范围选择

根据搜索内容类型选择 `timeRange`：

| 场景 | 推荐值 | 示例 |
|------|--------|------|
| 最新bug/安全漏洞 | `OneDay` 或 `OneWeek` | `"CVE-2024 漏洞修复"` |
| 快速迭代的技术 | `OneMonth` | `"React 19 新特性"` |
| 年度总结/版本更新 | `OneYear` | `"Python 3.12 特性 2024"` |
| 经典知识/基础概念 | `NoLimit` | `"TCP三次握手原理"` |
| 技术演进/发展历史 | `NoLimit` | `"Transformer发展历史"` |

### 搜索策略

#### 标准流程

```
1. search_web 获取多个来源摘要
   ↓
2. 筛选高质量结果（官方文档、权威博客、最新内容）
   ↓
3. fetch_content 深入获取重要链接的详细内容
   ↓
4. 综合多个来源，用自己的话重新表述
   ↓
5. 按参考链接规范添加参考资料
```

#### 高质量结果判断

**核心标准**（按优先级）：
1. **内容准确性** - 详实清晰、代码可运行、无技术错误
2. **实践价值** - 有可操作示例、真实场景案例
3. **深度与完整性** - 系统讲解、涵盖关键细节和陷阱
4. **时效性** - 技术内容优先最新的
5. **来源权威性** - 官方文档、知名平台（辅助参考）
6. **表达易懂性** - 通俗易懂、有类比图解（加分项）

---

### 搜索数量要求

⚠️ **重要**：每次搜索必须获取足够多的相关资料，**越多越好**。

**具体要求**：
- 不要满足于前3-5个搜索结果
- 应该浏览和提取**至少10个以上**的相关来源
- 对于重要技术主题，建议收集**15-20个**或更多资料
- 从多个平台获取信息（官方文档、技术博客、社区讨论、视频教程等）
- 确保覆盖不同的视角和解释方式

**原因**：
- 更多的资料 = 更全面的信息覆盖
- 可以交叉验证准确性
- 能够找到最佳实践和最权威的来源
- 避免单一来源的偏见或错误

---

### 搜索注意事项

1. **搜索词要具体**：避免宽泛词汇
2. **时间范围要合理**：不盲目使用最新时间
3. **多渠道验证**：重要信息交叉验证
4. **内容复述**：不直接复制，用自己的话表述
5. **链接格式统一**：严格按参考链接规范
6. **质量优先**：宁缺毋滥

## 第二部分：参考链接规范

### 参考资料添加要求

在文档中引用搜索结果时，**必须在引用处（内容紧跟其后）添加所有相关的参考资料链接**。

**格式要求**：
- 如果有高质量的参考资料，全部添加
- 如果没有高质量的，则找出最好的（拔尖的）添加
- **紧跟引用内容后面**（内容写完后下一行就是参考资料），不要放在章节末尾

---

### 高质量资料判断标准

判断参考资料质量时，应**聚焦于内容本身**，而非过度依赖平台权威性。平台可作为辅助参考，但不是决定因素。

#### 1. 内容准确性（核心）
- 内容详实、表达清晰，不是泛泛而谈
- 有据可查的观点，代码示例可运行
- 逻辑严密，没有明显的技术错误
- 深入讲解原理，而非仅停留在表面

#### 2. 实践价值（核心）
- 能解决实际问题，有可操作的示例
- 提供了真实场景的应用案例
- 对读者有启发性和指导意义
- 代码和配置经过验证，可以直接使用

#### 3. 深度与完整性（核心）
- 系统性地讲解知识点，不是碎片化信息
- 涵盖了关键细节和常见陷阱
- 提供了原理分析和最佳实践
- 有清晰的示例和运行结果

#### 4. 时效性（重要）
- 对于技术类内容，优先选择最新的
- 注意区分内容是否过时（如旧版本 API、已废弃的方案）
- 技术演进类内容应明确标注时间背景

#### 5. 来源权威性（辅助参考）
- 知名学术平台、官方文档、权威出版物可作为加分项
- 但不应以此排斥小众平台的优质内容
- 重点仍然看内容本身的质量，而非发布在哪里

#### 6. 表达易懂性（加分项）
- 能把复杂的概念用通俗易懂的语言解释清楚
- 使用生动的类比、图解或生活化示例帮助理解
- 对难点和易错点有专门的解释和提醒
- 循序渐进，适合目标读者群体的理解水平
- 避免堆砌术语，用简洁的语言讲透本质

---

### 链接格式规范

#### 统一格式要求

- 标题和平台名之间必须使用 ` -- `（两边各一个空格）
- 禁止使用单个 `-`、`—` 或 `---`
- 平台名必须统一且准确

**单个链接**：
```markdown
- [标题 -- 平台名](URL)
```
例如：`- [激活函数原理 -- CSDN](https://blog.csdn.net/...)`

**多个链接**：
```markdown
内容段落（联网搜索后编写的内容）...

（此处填写具体知识点名称）参考资料：
- [标题1 -- 平台名](URL1)
- [标题2 -- 平台名](URL2)
```

#### 正确示例

```markdown
- [Attention Is All You Need 论文原文 -- arXiv](https://arxiv.org/abs/1706.03762) ⭐值得阅读
- [Transformer模型的核心机制 -- 飞书文档](https://docs.feishu.cn/...)
- [PyTorch官方文档 -- PyTorch](https://pytorch.org/docs/...)
```

#### 错误示例

```markdown
- [标题-平台名](URL)          ❌ 缺少空格
- [标题 - 平台名](URL)        ❌ 使用单横线
- [标题---平台名](URL)        ❌ 使用三横线
- [标题 -- 平台名](URL)       ✅ 正确
```

---

### 高质量链接特别标注

对于质量特别高的参考资料（符合上述高质量标准），应在链接旁边添加 `⭐值得阅读` 标注，帮助读者快速识别重点内容。

```markdown
- [标题 -- 平台名](URL) ⭐值得阅读
```

**示例**：
```markdown
- [The Illustrated Transformer -- Jay Alammar](https://jalammar.github.io/illustrated-transformer/) ⭐值得阅读
```

⚠️ **注意**：`⭐值得阅读` 前面必须有一个空格。

---

### 标点符号统一

- 标题中的冒号使用中文冒号 `：`
- 标题中的括号使用中文括号 `（）`
- 英文专有名词保持原样（如 PyTorch、Transformer、BERT等）

**正确示例**：
```markdown
- [从零实现Transformer：第 2 部分 -- 缩放点积注意力 -- CSDN](URL)
- [PyTorch官方scaled_dot_product_attention文档 -- PyTorch](URL)
- [SDPA（Scaled Dot-Product Attention）详解 -- CSDN](URL)
```

### 文档内部引用

#### 文档间引用

> **注意**：每个文档文件夹下都有一个专门记录文档链接的文件（如 `00-Transformer技术专栏链接目录.md`），添加相关文档链接时需要去该文件查询。

- **文档内引用其他文档**：当涉及其他文档的知识点时，必须附上链接
  - 示例：`在[08a. 检索算法与策略-稠密检索技术](https://juejin.cn/...)([CSDN](https://blog.csdn.net/...))中我们学习了...`
  - 跨文档引用必须具体到章节，方便读者定位
  - 链接格式：掘金链接 / CSDN链接

3. 所有引用必须添加参考资料链接
