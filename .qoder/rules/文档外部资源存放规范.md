# 文档外部资源存放规范 📁

## 核心原则

每篇文档引用的本地外部资源（代码文件、图片、数据文件等）必须按 **文档编号** 归类存放，不得散落在项目各处以避免混乱。

## 目录结构

```
{文档所在父文件夹}/
├── resources/                          ← 资源根目录，所有文档的外部资源统一存放于此
│   ├── {文档编号}/                     ← 按文档编号归类，与文档文件名前缀一致
│   │   ├── code/                       ← 代码文件（如 .py, .js, .java, .sql 等）
│   │   ├── photo/                      ← 图片文件（如 .png, .jpg, .svg 等）
│   │   └── ...                         ← 可根据需要扩展其他子目录（如 data/, config/ 等）
│   └── ...
└── ...
```


## 路径组成规则

| 路径段 | 说明 | 取值示例 |
|--------|------|---------|
| **文档所在父文件夹** | 文档所在专题目录 | `Python-基础技术`、`Transformer`、`Java-JPA持久化技术` |
| **resources** | 固定资源目录名，统一使用小写英文 | `resources`（不可变更） |
| **文档编号** | 与文档文件名前缀编号一致 | `P2L`、`01`、`J6A` |
| **code** | 代码文件子目录 | `code` |
| **photo** | 图片文件子目录 | `photo` |

## 资源引用方式

在文档中引用这些资源时，使用**相对路径**从文档所在位置指向 resources 目录：

```markdown
<!-- 图片引用 -->
![图片描述](resources/P2L/photo/p2l_explode_comparison.png)

<!-- 代码文件引用 -->
参考完整代码：[pie_chart_generator.py](resources/P2L/code/pie_chart_generator.py)
```

## 新文档创建流程

创建新文档时，按照以下步骤建立资源目录：

```
1. 确定文档编号（如 P2L）
2. 在 resources/ 下创建 {文档编号}/ 目录
3. 在 {文档编号}/ 下创建 code/ 和 photo/ 子目录
4. 文档中引用的所有外部资源均存放于对应子目录
```

## 注意事项

- ❌ **禁止**：将资源文件直接放在文档同级目录下，造成目录混乱
- ❌ **禁止**：将资源散落在项目各处无统一管理
- ✅ **必须**：先建立 resources/{文档编号}/ 目录，再存放资源文件
- ✅ **必须**：资源的命名遵循对应专题规范（如图片命名遵循 `Python命令行绘图规范.md`）
- ✅ 如果文档没有外部资源，则无需创建 resources 目录
