# 参考资料规范

## 联网搜索后的参考资料

在文档中引用搜索结果时，必须在引用处（内容紧跟其后）添加所有相关的参考资料链接。

### 格式要求

- 如果有高质量的参考资料，全部添加
- 如果没有高质量的，则找出最好的（拔尖的）添加
- **紧跟引用内容后面**（内容写完后下一行就是参考资料），不要放在章节末尾

### 高质量资料判断标准

判断参考资料质量时，应**聚焦于内容本身**，而非过度依赖平台权威性。平台可作为辅助参考，但不是决定因素。

**1. 内容准确性（核心）**
- 内容详实、表达清晰，不是泛泛而谈
- 有据可查的观点，代码示例可运行
- 逻辑严密，没有明显的技术错误
- 深入讲解原理，而非仅停留在表面

**2. 实践价值（核心）**
- 能解决实际问题，有可操作的示例
- 提供了真实场景的应用案例
- 对读者有启发性和指导意义
- 代码和配置经过验证，可以直接使用

**3. 深度与完整性（核心）**
- 系统性地讲解知识点，不是碎片化信息
- 涵盖了关键细节和常见陷阱
- 提供了原理分析和最佳实践
- 有清晰的示例和运行结果

**4. 时效性（重要）**
- 对于技术类内容，优先选择最新的
- 注意区分内容是否过时（如旧版本 API、已废弃的方案）
- 技术演进类内容应明确标注时间背景

**5. 来源权威性（辅助参考）**
- 知名学术平台、官方文档、权威出版物可作为加分项
- 但不应以此排斥小众平台的优质内容
- 重点仍然看内容本身的质量，而非发布在哪里

### 链接格式

**统一格式要求**：
- 标题和平台名之间必须使用 ` -- `（两边各一个空格）
- 禁止使用单个 `-`、`—` 或 `---`
- 平台名必须统一且准确

单个链接：
```markdown
- [标题 -- 平台名](URL)
```
例如：`- [激活函数原理 -- CSDN](https://blog.csdn.net/...)`

多个链接：
```markdown
内容段落（联网搜索后编写的内容）...

（此处填写具体知识点名称）参考资料：
- [标题1 -- 平台名](URL1)
- [标题2 -- 平台名](URL2)
```

**正确示例**：
```markdown
- [Attention Is All You Need 论文原文 -- arXiv](https://arxiv.org/abs/1706.03762) ⭐值得阅读
- [Transformer模型的核心机制 -- 飞书文档](https://docs.feishu.cn/...)
- [PyTorch官方文档 -- PyTorch](https://pytorch.org/docs/...)
```

**错误示例**：
```markdown
- [标题-平台名](URL)          ❌ 缺少空格
- [标题 - 平台名](URL)        ❌ 使用单横线
- [标题---平台名](URL)        ❌ 使用三横线
- [标题 -- 平台名](URL)       ✅ 正确
```

### 高质量链接特别标注

对于质量特别高的参考资料（符合上述4条高质量标准），应在链接旁边添加 `⭐值得阅读` 标注，帮助读者快速识别重点内容。

```markdown
- [标题 -- 平台名](URL) ⭐值得阅读
```

示例：
```markdown
- [The Illustrated Transformer -- Jay Alammar](https://jalammar.github.io/illustrated-transformer/) ⭐值得阅读
```

**注意**：`⭐值得阅读` 前面必须有一个空格。

### 标点符号统一

- 标题中的冒号使用中文冒号 `：`
- 标题中的括号使用中文括号 `（）`
- 英文专有名词保持原样（如 PyTorch、Transformer、BERT等）

**正确示例**：
```markdown
- [从零实现Transformer：第 2 部分 -- 缩放点积注意力 -- CSDN](URL)
- [PyTorch官方scaled_dot_product_attention文档 -- PyTorch](URL)
- [SDPA（Scaled Dot-Product Attention）详解 -- CSDN](URL)
```

## 文档内部引用

### 文档间引用

> **注意**：每个文档文件夹下都有一个专门记录文档链接的文件（如 `00-Transformer技术专栏链接目录.md`），添加相关文档链接时需要去该文件查询。

- **文档内引用其他文档**：当涉及其他文档的知识点时，必须附上链接
  - 示例：`在[08a. 检索算法与策略-稠密检索技术](https://juejin.cn/...)([CSDN](https://blog.csdn.net/...))中我们学习了...`
  - 跨文档引用必须具体到章节，方便读者定位
  - 链接格式：掘金链接 / CSDN链接

## 引用规则

- 不可以直接复制他人的语句，要用自己的话重新表述内容
- 如果需要引用内容，必须按照 MLA 格式来标注
