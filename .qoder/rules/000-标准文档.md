## 2. 手动实现缩放点积注意力 🧮 / Manual Implementation of Scaled Dot-Product Attention

> 📖 **Note:** 本节从零编写缩放点积注意力的完整代码，逐行讲解 / This section writes complete scaled dot-product attention code from scratch with line-by-line explanations.

### 2.1 核心公式回顾 📝 / Core Formula Review

在写代码之前，先回顾一下缩放点积注意力（Scaled Dot-Product Attention）的核心公式：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{Q \times K^T}{\sqrt{d_k}}\right) \times V
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

这个公式可以拆解为四个步骤：📝

1. 计算 Q 和 K 的 Transpose 的 Dot Product → 得到 Attention Scores
2. 除以 $\sqrt{d_k}$ 进行 Scaling → 防止数值过大
3. Softmax 进行 Normalization → 将 Scores 转换为 Probability Distribution（Attention Weights）
4. 用 Weights 对 V 做 Weighted Sum → 得到最终 Output

**什么是点积（Dot Product）？** 🤔

点积是两个向量之间的一种运算，结果是一个标量（数值）。对于两个 n 维向量 $\mathbf{a} = [a_1, a_2, \ldots, a_n]$ 和 $\mathbf{b} = [b_1, b_2, \ldots, b_n]$，它们的点积定义为：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\mathbf{a} \cdot \mathbf{b} = a_1 b_1 + a_2 b_2 + \cdots + a_n b_n = \sum_{i=1}^{n} (a_i \times b_i)
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

**Dot Product 的几何意义** 📐：可以衡量两个向量的相似程度

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\mathbf{a} \cdot \mathbf{b} = |\mathbf{a}| \times |\mathbf{b}| \times \cos(\theta)
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

- $|\mathbf{a}|$ 和 $|\mathbf{b}|$ 是向量的长度（模）
- $\theta$ 是两个向量之间的夹角
- **$\cos(\theta)$ 越大，说明两个向量方向越接近，越相似**

**直观类比** 🎨：想象点积是在计算"两个人的兴趣爱好匹配度"
- 如果两个人都喜欢同样的事物（方向相同），点积结果很大 → 高度匹配
- 如果两个人兴趣完全不同（方向垂直），点积结果接近 0 → 没有共同点
- 如果两个人爱好完全相反（方向相反），点积结果为负 → 互相抵触

在 Attention 机制中，我们使用 Dot Product 来计算 Query 和 Key 的 Similarity：🔍
- Dot Product 结果越大 → 两个向量越 Similar → Attention Weight 越高
- Dot Product 结果越小（甚至为负）→ 两个向量越不相关 → Attention Weight 越低

💡 这就是为什么用 Dot Product 来计算 Attention Scores——它能直观地反映 Query 和 Key 的匹配程度。

### 2.2 完整代码实现 💻 / Complete Code Implementation

👇 下面是基于 PyTorch 的完整手动实现：

```python
import torch                                              # 导入 PyTorch 核心库，提供张量运算 🔥
import torch.nn as nn                                     # 导入神经网络模块，包含 Dropout 等层 🧠
import math                                               # 导入数学库，用于计算 sqrt 🔢

"""缩放点积注意力机制的手动实现 💻 / Manual Implementation of Scaled Dot-Product Attention

参数 / Args:
    dropout: Dropout概率，用于防止过拟合（默认0.1）/ Dropout probability to prevent overfitting (default: 0.1)
    
示例 / Example:
    attention = ScaledDotProductAttention(dropout=0.1)
"""
class ScaledDotProductAttention(nn.Module):
    """初始化 Dropout 层 🛠️ / Initialize Dropout Layer
    
    参数 / Args:
        dropout: Dropout概率（默认0.1），示例：0.1 表示训练时随机丢弃 10% 的权重 / Dropout probability (default: 0.1)
        
    返回 / Returns:
        无 / None
        
    示例 / Example:
        self.dropout = nn.Dropout(0.1)
    """
    def __init__(self, dropout=0.1):
        super(ScaledDotProductAttention, self).__init__()
        self.dropout = nn.Dropout(dropout)
    
    """前向传播计算缩放点积注意力 ⚡ / Forward Pass for Scaled Dot-Product Attention
    
    参数 / Args:
        Q: 查询矩阵 [batch_size=2, n_heads=4, seq_len_q=10, d_k=64] / Query matrix
        K: 键矩阵 [2, 4, 10, 64] / Key matrix
        V: 值矩阵 [2, 4, 10, 64] / Value matrix
        mask: 可选掩码矩阵 / Optional mask matrix
        
    返回 / Returns:
        output: 注意力输出 [2, 4, 10, 64] / Attention output
        attention_weights: 注意力权重 [2, 4, 10, 10] / Attention weights
        
    示例 / Example:
        output, weights = attention(Q, K, V, mask=None)
    """
    def forward(self, Q, K, V, mask=None):
        # 1️⃣ 获取 Q 的最后一个维度大小，即 d_k / Get the last dimension of Q, which is d_k
        # 示例 / Example: Q.shape=[2,4,10,64] → d_k=64
        d_k = Q.size(-1)
        
        # 2️⃣ 计算 Q 和 K^T 的点积，并除以 sqrt(d_k) 进行缩放 / Compute Q·K^T and scale by 1/√d_k
        # 数据流动 / Data flow: Q[2,4,10,64] × K^T[2,4,64,10] → scores[2,4,10,10] / sqrt(64) → scores[2,4,10,10]
        scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(d_k)
        
        # 3️⃣ 如果提供了掩码，将掩码为0的位置填充为负无穷 🎭 / Apply mask: fill 0 positions with -inf
        # 数据流动 / Data flow: scores[2,4,10,10] + mask → scores[2,4,10,10] (some positions become -inf)
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float('-inf'))
        
        # 4️⃣ 对分数应用 Softmax，得到注意力权重 📊 / Apply Softmax to get attention weights
        # 数据流动 / Data flow: scores[2,4,10,10] → Softmax → attention_weights[2,4,10,10] (each row sums to 1)
        attention_weights = torch.softmax(scores, dim=-1)
        
        # 5️⃣ 应用 Dropout（训练时随机丢弃部分权重，防止过拟合） 🎲 / Apply Dropout (randomly zero out weights during training)
        # 数据流动 / Data flow: attention_weights[2,4,10,10] → Dropout → attention_weights[2,4,10,10] (some become 0)
        attention_weights = self.dropout(attention_weights)
        
        # 6️⃣ 用注意力权重对 V 做加权求和 ⚖️ / Weighted sum of V using attention weights
        # 数据流动 / Data flow: attention_weights[2,4,10,10] × V[2,4,10,64] → output[2,4,10,64]
        output = torch.matmul(attention_weights, V)
        
        return output, attention_weights
```

### 2.3 代码逐行解析 🔍 / Line-by-Line Code Analysis

> 📖 本节详细拆解每一步的计算过程和数据形状变化

**第1步：获取维度** 1️⃣

```python
d_k = Q.size(-1)
```

`Q.size(-1)` 获取 Q 的最后一个 Dimension，也就是每个 Head 的 Dimension $d_k$。这个值用来计算 Scaling Factor $\sqrt{d_k}$。

**Head Dimension $d_k$** 是什么？🧠

在 Multi-Head Attention 中，模型会将输入向量的总 Dimension $d_{\text{model}}$ 平均分配给 $h$ 个 Attention Heads。因此：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
d_k = \frac{d_{\text{model}}}{h}
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

举个例子，在标准 Transformer（如 BERT-base）中：
- $d_{\text{model}} = 512$（Model 总 Dimension）
- $h = 8$（8 个 Attention Heads）
- 每个 Head 的 Dimension $d_k = 512 / 8 = 64$

这样每个 Head 独立处理 64 Dimension 的信息，8 个 Head 并行计算，最后将结果 Concatenate 起来，既保证了计算效率，又能从不同的"视角"捕捉信息。👀

**为什么用 `-1`？** 🤔

在 PyTorch 中，`.size()` 支持 Negative Indexing，`-1` 表示最后一个 Dimension，`-2` 表示倒数第二个 Dimension，以此类推。这与 Python List 的 Negative Indexing 语义一致。

假设 Q 的 Shape 是 `[batch_size, n_heads, seq_len, d_k]`：

| 写法 | 等价写法 | 获取值 |
|------|---------|--------|
| `Q.size(0)` | - | batch_size |
| `Q.size(1)` | - | n_heads |
| `Q.size(2)` | `Q.size(-2)` | seq_len |
| `Q.size(3)` | `Q.size(-1)` | **d_k** |

💪 使用 `-1` 的好处是代码更 Robust——即使 Tensor Dimension 增加（比如从 4 维变成 5 维），`-1` 始终指向最后一个 Dimension，不需要修改代码。

**第2步：计算点积并缩放** 2️⃣

```python
scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(d_k)
```

- `K.transpose(-2, -1)`：将 K 的最后两个 Dimension 交换，实现 $K^T$（Transpose）
- `torch.matmul(...)`：执行 Matrix Multiplication $Q \times K^T$
- `/ math.sqrt(d_k)`：除以 $\sqrt{d_k}$ 进行 Scaling

**为什么要对 K 进行 Transpose？** 🤔

Attention 机制的核心是计算 **每个 Query 与所有 Key 的 Similarity**。数学上，这需要计算 Q 和 K 的 Dot Product：

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\text{Scores} = Q \times K^T
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

如果不 Transpose，直接计算 $Q \times K$：
- Q Shape：$[\text{batch}, \text{heads}, \text{seq\_len}, d_k] = [2, 4, 10, 64]$
- K Shape：$[\text{batch}, \text{heads}, \text{seq\_len}, d_k] = [2, 4, 10, 64]$
- Matrix Multiplication 要求：第一个 Matrix 的 Column 数 = 第二个 Matrix 的 Row 数
- 这里 $64 \neq 10$，无法直接相乘

对 K Transpose 后：
- $K^T$ Shape：$[\text{batch}, \text{heads}, d_k, \text{seq\_len}] = [2, 4, 64, 10]$
- 现在 Q 的最后一个 Dimension $64$ = $K^T$ 的倒数第二个 Dimension $64$，可以相乘
- 结果 Shape：$[\text{batch}, \text{heads}, \text{seq\_len}, \text{seq\_len}] = [2, 4, 10, 10]$

**直观类比** 🧩：为什么要 Transpose？就像拼图时需要旋转其中一块才能拼接
- Q 是"Query Token List"（10 个 Token，每个 64 Dimension）
- K 是"Key Token List"（10 个 Token，每个 64 Dimension）
- 要计算"每个 Query Token"与"所有 Key Token"的 Similarity，需要把 K 旋转一下
- Transpose 后，Q 的 64 Dimension 才能和 K 的 64 Dimension"对齐"，完成——匹配

这个 `[10, 10]` 的 Matrix 就是 **Attention Scores Matrix** 🎯，其中第 i 行第 j 列的值表示： **第 i 个 Position 的 Token 对第 j 个 Position 的 Token 的 Attention Weight**。

举个例子 🌰，如果 scores[2][5] = 3.5，表示句子中第 3 个词（索引2）对第 6 个词（索引5）有较强的注意力。

**第3步：应用掩码** 3️⃣

```python
if mask is not None:                                                      # 判断是否提供了掩码，示例：mask.shape=[2,4,10,10]
    scores = scores.masked_fill(mask == 0, float('-inf'))                 # 将掩码为0的位置填充为 -inf，数据流动：scores[2,4,10,10] → scores[2,4,10,10]（某些位置变为 -inf）
```

Mask 的作用是把某些 Position 的 Scores 变成负无穷。经过 Softmax 后，$e^{-\infty} = 0$，所以这些 Position 的 Attention Weights 会变成 0，相当于"屏蔽"了这些 Position。🚫

**直观类比** 🎭：Mask 就像考试时"跳过不会的题目"
- Padding Mask：句子太短用 `<PAD>` 填充 → 这些 Position 是"空白题"，直接跳过不算分
- Causal Mask：Decoder 只能看到前面的 Token → 就像"只能做前面的题，不能偷看后面的题"

这在两种场景特别重要：⚠️
- **Padding Mask**：句子长度不一，用 `<PAD>` 填充的 Position 需要屏蔽
- **Causal Mask**：Decoder 里，当前 Token 只能看到前面的 Token，不能看到后面的 Token

**第4步：Softmax 归一化** 4️⃣

```python
attention_weights = torch.softmax(scores, dim=-1)                         # 在最后一个维度做 Softmax 归一化，数据流动：scores[2,4,10,10] → attention_weights[2,4,10,10]（每行和为1）
```

`dim=-1` 表示在最后一个 Dimension 上做 Softmax。对于 `[2, 4, 10, 10]` 的 Tensor，就是在每个 `[10, 10]` Matrix 的每一 Row 上做 Normalization，让每行加起来等于 1。

**为什么要用 Softmax 进行 Normalization？** 🤔

Softmax 在 Attention 机制中有三个关键作用：

1. **转化为 Probability Distribution** 📊
   
   Attention Scores（Dot Product 结果）可以是任意 Real Number（正数、负数、很大或很小），但我们需要的是 **Weights** ——表示"应该关注多少"。Softmax 将 Scores 转换为 `[0, 1]` 区间的值，且所有 Weights 之和为 1，形成合法的 Probability Distribution。

2. **放大差异，突出重点（Amplify Differences）** 🔍
   
   Softmax 使用指数函数 $e^x$，具有"放大差异"的特性：
   - Scores 较高的会得到更高的 Weights
   - Scores 较低的会被压制得更低
   
   例如，Scores $[1.0, 2.0, 3.0]$ 经过 Softmax 后变成 $[0.09, 0.24, 0.67]$，最高的 3.0 获得了 67% 的 Weight，帮助 Model 清晰聚焦最重要的信息。

3. **消除负数，保证正向加权（Ensure Positive Weights）** ✅
   
   Dot Product 结果可能为负（当两个向量方向相反时）。如果直接用负数加权，会抵消 Value 中的有用信息。Softmax 的指数函数 $e^x$ 始终输出正数，确保所有 Weights 都是"正向贡献"，仅通过大小区分关注程度。

**直观类比** 🗳️：想象 Softmax 是一个"投票计数器"——它把每个人的"支持度打分"转换成"得票百分比"，让模型能够按比例综合所有人的意见。


- attention_weights Shape: `[2, 4, 10, 10]`
- V Shape: `[2, 4, 10, 64]`
- Output Shape: `[2, 4, 10, 64]`

**什么是 Weighted Sum？** 🤔

💡 Weighted Sum 是 Attention 机制的最后一步，也是核心思想的体现： **根据 Attention Weights，将 Value Vectors 按 Importance 进行 Aggregate**。

数学上，对于第 i 个位置的输出：📐

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\text{output}[i] = \sum_{j=1}^{n} \text{attention\_weights}[i][j] \times V[j]
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

其中：📝
- $\text{attention\_weights}[i][j]$ 表示第 i 个 Token 对第 j 个 Token 的 Attention Weight
- $V[j]$ 是第 j 个 Position 的 Value Vector（包含实际 Semantic Information）
- Summation 遍历所有 Position j

**直观理解** 🎯：想象你在写一篇关于"猫"的文章 ✍️，Model 已经计算出：
- "猫"对"小猫"的 Weight 是 0.6 🐱
- "猫"对"动物"的 Weight 是 0.3 🐾
- "猫"对"喜欢"的 Weight 是 0.1 ❤️

那么"猫"的最终 Representation 就是：💭

<!-- 数学公式必须使用 LaTeX 格式 -->
$$
\text{output}[\text{猫}] = 0.6 \times V[\text{小猫}] + 0.3 \times V[\text{动物}] + 0.1 \times V[\text{喜欢}]
$$
<!-- 数学公式必须使用 LaTeX 格式 -->

🌟 这样，"猫"的新 Representation 就融合了"小猫"、"动物"、"喜欢"的 Semantic Information，且根据 Relevance 分配了不同的比重。

**为什么叫"Weighted"Sum？** 🤔

普通的 Sum 是简单相加：`a + b + c` ➕

Weighted Sum 是给每个 Item 分配一个 Weight：`0.6a + 0.3b + 0.1c` ⚖️

Weight 的意义在于：🎯
- **Weight 高** ⬆️ → 该 Item 对结果影响大
- **Weight 低** ⬇️ → 该 Item 对结果影响小
- **Weight 为 0** 🚫 → 该 Item 被完全忽略

这正是 Attention 机制"抓重点"的本质—— **重要的信息给高 Weight，无关的信息给低 Weight**。🎯

写好了代码，我们来看看注意力权重到底长什么样。下面是一个简单的可视化示例：

```python

# 设置 Matplotlib 支持中文显示 🇨🇳

"""手动实现的缩放点积注意力机制

参数:
    dropout: Dropout概率，用于防止过拟合（默认0.1）
    
示例:
    attention = ScaledDotProductAttention(dropout=0.1)
"""
class ScaledDotProductAttention(torch.nn.Module):
    """初始化 Dropout 层
    
    参数:
        dropout: Dropout概率（默认0.1），示例：0.1 表示训练时随机丢弃 10% 的权重
        
    返回:
        无
        
    示例:
        self.dropout = torch.nn.Dropout(0.1)
    """
    def __init__(self, dropout=0.1):
        super(ScaledDotProductAttention, self).__init__()
        self.dropout = torch.nn.Dropout(dropout)
    
    """前向传播计算缩放点积注意力
    
    参数:
        Q: 查询矩阵 [batch=2, heads=4, seq_len_q=5, d_k=64]
        K: 键矩阵 [2, 4, 5, 64]
        V: 值矩阵 [2, 4, 5, 64]
        mask: 可选掩码
        
    返回:
        output: 注意力输出 [2, 4, 5, 64]
        attention_weights: 注意力权重 [2, 4, 5, 5]
        
    示例:
        output, weights = attention(Q, K, V, mask=None)
    """
    def forward(self, Q, K, V, mask=None):
        # 1. 获取维度 d_k，示例：Q.shape=[2,4,5,64] → d_k=64
        d_k = Q.size(-1)
        # 2. 计算 Q×K^T/√d_k，数据流动：Q[2,4,5,64] × K^T[2,4,64,5] → scores[2,4,5,5]
        scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(d_k)
        # 3. 应用掩码（如有）
        if mask is not None:                                                      # 判断是否提供掩码
            scores = scores.masked_fill(mask == 0, float('-inf'))                 # 填充 -inf，数据流动：scores[2,4,5,5] → scores[2,4,5,5]（某些位置变为 -inf）
        # 4. Softmax 归一化，数据流动：scores[2,4,5,5] → attention_weights[2,4,5,5]（每行和为1）
        attention_weights = torch.softmax(scores, dim=-1)
        # 5. Dropout，数据流动：attention_weights[2,4,5,5] → attention_weights[2,4,5,5]（部分变为0）
        attention_weights = self.dropout(attention_weights)
        # 6. 加权求和，数据流动：attention_weights[2,4,5,5] × V[2,4,5,64] → output[2,4,5,64]
        output = torch.matmul(attention_weights, V)
        return output, attention_weights
```

```python
    V = torch.randn(batch_size, n_heads, seq_len, d_v)                            # 值矩阵 V：[2,4,5,64] 💎
    
    # 创建注意力模块，dropout=0.0 表示不丢弃权重 🛠️
    attention = ScaledDotProductAttention(dropout=0.0)
    
    # 前向传播，数据流动：Q[2,4,5,64], K[2,4,5,64], V[2,4,5,64] → output[2,4,5,64], weights[2,4,5,5] ⚡
    output, weights = attention(Q, K, V)
    
    print("=" * 50)                                                         # 打印分隔线 📏
    print("缩放点积注意力测试")                                              # 打印测试标题 📝
    print("=" * 50)                                                         # 打印分隔线 📏
    print(f"Q 形状: {Q.shape}")                                             # 打印 Q 的形状，示例：[2,4,5,64]
    print(f"K 形状: {K.shape}")                                             # 打印 K 的形状，示例：[2,4,5,64]
    print(f"V 形状: {V.shape}")                                             # 打印 V 的形状，示例：[2,4,5,64]
    print(f"输出形状: {output.shape}")                                      # 打印输出形状，示例：[2,4,5,64]
    print(f"注意力权重形状: {weights.shape}")                               # 打印权重形状，示例：[2,4,5,5]
    print(f"权重每行求和: {weights[0, 0].sum(dim=-1)}")                     # 验证权重归一化，每行和应接近 1.0 ✅
    print("=" * 50)                                                         # 打印分隔线 📏
    
    return output, weights                                                  # 返回输出和权重


"""可视化注意力权重热力图 👁️

返回:
    无（保存图片到文件）
    
示例:
    visualize_attention(weights, tokens=['我', '喜欢', '学习'])
"""
def visualize_attention(attention_weights, tokens=None):
    # 导入 numpy 用于张量转换 🔢
    import numpy as np
    
    # 如果是多维张量，取第一个 batch 和第一个 head 🎯
    if attention_weights.dim() == 4:                                        # 判断是否为 4 维张量 [batch, heads, seq_q, seq_k]
        attention_weights = attention_weights[0, 0]                         # 取 [0,0]，数据流动：[2,4,5,5] → [5,5]
    elif attention_weights.dim() == 3:                                      # 判断是否为 3 维张量 [heads, seq_q, seq_k]
        attention_weights = attention_weights[0]                            # 取 [0]，数据流动：[4,5,5] → [5,5]

if __name__ == "__main__":
```
